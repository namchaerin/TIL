# 자바스크립트 ES6+

## 1. 학습내용

### **[클래스 (Class)]**
* **객체 설계도**: ES6에서 도입된 클래스는 객체의 상태를 설정하는 `constructor`와 동작을 정의하는 메서드로 구성된다.
* **메모리 효율성**: 클래스 내부에 정의된 메서드는 인스턴스마다 복사되지 않고 프로토타입에 저장되어 모든 인스턴스가 공유한다.

### **[화살표 함수 (Arrow Function)]**
* **간결한 문법**: `function` 키워드 대신 `=>`를 사용하며, 본문이 한 줄일 경우 중괄호와 `return`을 생략할 수 있다.
* **this 바인딩**: 일반 함수와 달리 자신만의 `this`를 가지지 않으며, 함수가 선언된 상위 스코프의 `this`를 그대로 계승한다.

### **[이터러블 (Iterable)]**
* **순회 규약**: 배열, 문자열 등 `for...of` 문으로 순회 가능한 객체는 이터러블 프로토콜을 따른다.
* **배열 변환**: 인덱스와 `length`만 가진 유사 배열 객체는 `Array.from()`을 통해 진짜 배열로 변환하여 배열 메서드를 사용할 수 있다.

### **[Rest 파라미터와 Spread 문법]**
* **Rest 파라미터**: 함수의 매개변수에서 `...args` 형식을 사용하여 정해지지 않은 수의 인수를 하나의 배열로 모은다.
* **Spread 문법**: 배열이나 객체 리터럴 내부에서 `...`을 사용해 요소들을 개별 값으로 펼쳐서 복사하거나 병합한다.

### **[구조 분해 할당 (Destructuring)]**
* **배열 구조 분해**: 배열의 요소를 순서에 따라 개별 변수에 할당하며, 쉼표를 통해 특정 요소를 건너뛰거나 기본값을 설정한다.
* **객체 구조 분해**: 객체의 키(key) 이름을 기준으로 값을 추출하며, 콜론(`:`)을 사용하여 새로운 변수명으로 할당한다.

## 2. 느낀 점
ES6 이후 도입된 문법들이 코드의 길이를 획기적으로 줄여줄 뿐만 아니라, `this` 바인딩이나 프로토타입 공유와 같은 자바스크립트의 핵심 동작 원리를 더욱 명확하게 해준다는 점을 느꼈다.

## 3. 활용 방안
복잡한 객체나 배열 데이터를 다룰 때 구조 분해 할당과 스프레드 문법을 적극 사용하여 가독성을 높이고, 화살표 함수를 활용해 간결한 콜백 로직을 구현한다.
