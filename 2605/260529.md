# Spring Boot REST API와 Jackson JSON 처리

## 1. Jackson JSON
- Jackson은 Java 객체를 JSON으로 변환하거나 JSON 데이터를 Java 객체로 변환하는 라이브러리이다.
- Spring Boot에서는 기본적으로 Jackson 라이브러리를 사용하여 JSON 직렬화/역직렬화를 처리한다.

| 용어 | 설명 |
| --- | --- |
| 직렬화(Serialization) | Java 객체 → JSON 변환 |
| 역직렬화(Deserialization) | JSON → Java 객체 변환 |

---

## 2. REST Controller

### @RestController 역할

```java
@RestController
@RequestMapping("/api/v1/members")
public class JacksonController {
}
```

- `@RestController`는 반환 데이터를 JSON 형태로 자동 변환한다.
- 내부적으로 `@ResponseBody`가 포함되어 있다.
- Spring이 Jackson의 `ObjectMapper`를 이용해 객체를 JSON으로 변환한다.

---

## 3. Java 객체 → JSON 변환

### 3-1. List 객체 반환

```java
@GetMapping
public ResponseEntity<List<MemberDTO>> findMembers() {
    return ResponseEntity.ok(memberList);
}
```

### 동작 방식

1. Controller가 `List<MemberDTO>` 반환
2. Spring이 Jackson 호출
3. Jackson이 List 객체를 JSON 배열로 변환
4. HTTP Response Body에 JSON 데이터 담아 응답

### JSON 응답 예시

```json
[
  {
    "no": 1,
    "name": "판다",
    "age": 5
  },
  {
    "no": 2,
    "name": "코알라",
    "age": 3
  }
]
```

---

### 3-2. 단일 객체 반환

```java
@GetMapping("/{memberNo}")
public ResponseEntity<MemberDTO> findMember(@PathVariable int memberNo) {
    return ResponseEntity.ok(foundMember);
}
```

### 특징

- `MemberDTO` 객체 1개를 JSON 객체 형태로 변환한다.
- `@PathVariable`로 URL 값을 전달받는다.

### JSON 응답 예시

```json
{
  "no": 1,
  "name": "판다",
  "age": 5
}
```

## 4. JSON → Java 객체 변환

### @RequestBody

```java
@PostMapping
public ResponseEntity<Map<String, Object>> createMember(
        @RequestBody MemberDTO member) {
}
```

### 동작 방식

1. 클라이언트가 JSON 요청 전송
2. Spring이 Jackson 호출
3. JSON 데이터를 `MemberDTO` 객체로 변환
4. Controller 메소드의 매개변수로 전달

### 요청 JSON 예시

```json
{
  "no": 4,
  "name": "토끼",
  "age": 2
}
```

### 핵심 개념

- `@RequestBody`는 HTTP Body의 JSON 데이터를 Java 객체로 변환한다.
- 역직렬화(Deserialization)가 수행된다.

---

## 5. ResponseEntity

### ResponseEntity 역할

```java
return ResponseEntity.status(HttpStatus.CREATED).body(response);
```

### 특징

- HTTP 상태 코드 설정 가능
- 응답 데이터(body) 설정 가능
- Header 제어 가능

| 메소드 | 설명 |
| --- | --- |
| ok() | 200 OK |
| status() | 상태 코드 지정 |
| body() | 응답 데이터 설정 |

---

## 6. Map 기반 JSON 응답

### Wrapper 응답 구조

```java
Map<String, Object> response = new LinkedHashMap<>();
response.put("message", "회원 목록 조회 성공");
response.put("count", memberList.size());
response.put("data", memberList);
```

### 특징

- 데이터 외 추가 정보 전달 가능
- API 응답 형식을 통일할 수 있다.
- 실무에서 많이 사용하는 방식이다.

### JSON 응답 예시

```json
{
  "message": "회원 목록 조회 성공",
  "count": 3,
  "data": [
    {
      "no": 1,
      "name": "판다"
    }
  ]
}
```

## 핵심 정리

- `@RestController`는 데이터를 JSON으로 반환한다.
- Spring Boot는 Jackson 라이브러리를 기본 사용한다.
- `@RequestBody`는 JSON → Java 객체 변환이다.
- ResponseEntity는 HTTP 응답을 제어한다.
- `ObjectMapper`는 JSON 변환 핵심 클래스이다.
- List는 JSON 배열로, 객체는 JSON 객체로 변환된다.
