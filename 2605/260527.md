# Spring Bean Scope / Life Cycle / Properties

## 1. Bean Scope

Bean Scope란 Spring Bean 객체의 생성 범위를 의미한다.

### Bean Scope 종류

| Scope | 설명 |
|---|---|
| singleton | 하나의 객체를 생성하여 공유 |
| prototype | 요청 시마다 새로운 객체 생성 |
| request | HTTP 요청마다 새로운 객체 생성 |
| session | HTTP 세션마다 새로운 객체 생성 |

---

# 1-1. Singleton Scope

Spring의 기본 Scope이다.

```java
@Bean
@Scope("singleton")
public ShoppingCart cart() {
    return new ShoppingCart();
}
```

기본값이므로 `@Scope("singleton")` 은 생략 가능하다.

```java
@Bean
public ShoppingCart cart() {
    return new ShoppingCart();
}
```

---

## 특징

- 객체를 하나만 생성
- 모든 곳에서 동일 객체 공유
- 메모리 효율이 좋음
- 상태값 저장 시 동시성 문제 발생 가능

---

## 예시

```java
ShoppingCart cart1 = context.getBean("cart", ShoppingCart.class);
ShoppingCart cart2 = context.getBean("cart", ShoppingCart.class);

System.out.println(cart1.hashCode());
System.out.println(cart2.hashCode());
```

### 결과

```text
111111
111111
```

동일한 객체를 사용한다.

---

# 1-2. Prototype Scope

Bean 요청 시마다 새로운 객체를 생성한다.

```java
@Bean
@Scope("prototype")
public ShoppingCart cart() {
    return new ShoppingCart();
}
```

---

## 특징

- 매번 새로운 객체 생성
- 상태 공유 문제 해결 가능
- 객체 생성 비용 증가

---

## 예시

```java
ShoppingCart cart1 = context.getBean("cart", ShoppingCart.class);
ShoppingCart cart2 = context.getBean("cart", ShoppingCart.class);

System.out.println(cart1.hashCode());
System.out.println(cart2.hashCode());
```

### 결과

```text
111111
222222
```

서로 다른 객체가 생성된다.

---

# Singleton vs Prototype

| 구분 | Singleton | Prototype |
|---|---|---|
| 객체 생성 수 | 1개 | 요청마다 생성 |
| 상태 공유 | O | X |
| 메모리 효율 | 높음 | 상대적으로 낮음 |
| 사용 예시 | Service, Repository | Cart, TempObject |

# 2. Bean Life Cycle

Spring Bean은 생성부터 소멸까지 생명주기를 가진다.

```text
객체 생성
→ 의존성 주입
→ 초기화(init)
→ 사용
→ 소멸(destroy)
```

---

# 2-1. initMethod / destroyMethod

## 클래스

```java
public class Owner {

    public void openShop() {
        System.out.println("가게 오픈");
    }

    public void closeShop() {
        System.out.println("가게 종료");
    }
}
```

---

## Bean 설정

```java
@Bean(initMethod = "openShop", destroyMethod = "closeShop")
public Owner owner() {
    return new Owner();
}
```

---

## 컨테이너 종료

```java
((AnnotationConfigApplicationContext) context).close();
```

---

## 실행 결과

```text
가게 오픈
...
가게 종료
```

---

# 2-2. @PostConstruct / @PreDestroy

Annotation 방식으로 Bean 생명주기를 관리할 수 있다.

---

## 코드 예시

```java
@Component
public class Owner {

    @PostConstruct
    public void openShop() {
        System.out.println("가게 오픈");
    }

    @PreDestroy
    public void closeShop() {
        System.out.println("가게 종료");
    }
}
```

---

## 특징

| 어노테이션 | 설명 |
|---|---|
| @PostConstruct | Bean 생성 후 실행 |
| @PreDestroy | Bean 소멸 전 실행 |

---

## 활용 예시

- DB Connection 초기화
- Redis 연결
- Kafka Producer 생성
- 캐시 로딩
- Thread 종료 처리

# 3. Properties

설정값을 외부 파일로 분리하여 관리하는 방식이다.

---

# 3-1. Properties 파일

## product-info.properties

```properties
bread.name=붕어빵
bread.price=1000

milk.name=딸기우유
milk.price=1500
milk.capacity=500
```

---

# 3-2. @PropertySource

```java
@Configuration
@PropertySource("classpath:product-info.properties")
public class ContextConfiguration {

}
```

---

# 3-3. @Value 사용

```java
@Value("${bread.name}")
private String breadName;

@Value("${bread.price}")
private int breadPrice;
```

---

## 기본값 설정

```java
@Value("${bread.price:0}")
private int breadPrice;
```

값이 존재하지 않을 경우 기본값 `0` 사용.

---

# 3-4. Bean 생성 시 사용

```java
@Bean
public Product bread() {
    return new Bread(breadName, breadPrice);
}
```

---

# Properties 장점

| 장점 | 설명 |
|---|---|
| 환경 분리 | dev / prod 분리 가능 |
| 유지보수 | 코드 수정 감소 |
| 보안 | 민감 정보 외부 관리 가능 |
