# Spring AOP (Aspect Oriented Programming)

## AOP란?

AOP(관점 지향 프로그래밍)는 핵심 비즈니스 로직과 공통 기능을 분리하여 관리하는 방식이다.

예를 들어:

- 로그 출력
- 권한 체크
- 트랜잭션 처리
- 실행 시간 측정

같은 기능은 여러 클래스에서 반복적으로 사용된다.

이런 공통 기능을 따로 분리하여 관리하는 것이 AOP이다.

---

## Aspect

공통 관심사를 모아둔 클래스이다.

```java
@Aspect
@Component
public class LoggingAspect {
}
```

| 어노테이션 | 역할 |
|---|---|
| @Aspect | AOP 기능을 수행하는 클래스 |
| @Component | Spring Bean 등록 |

---

## Pointcut

Advice를 어디에 적용할지 지정하는 조건이다.

```java
@Pointcut("execution(* com.ohgiraffers.section01.*Service.*(..))")
public void logPointcut() {
}
```

---

### execution 표현식 해석

```text
execution(
    *                                   // 리턴 타입
    com.ohgiraffers.section01           // 패키지
    .*Service                           // 클래스명
    .*                                  // 메서드명
    (..)                                // 매개변수
)
```

---

### 의미

```text
com.ohgiraffers.section01 패키지 안에서
이름이 Service로 끝나는 클래스의
모든 메서드 실행 지점
```

에 Advice를 적용한다.

---

## AOP 핵심 용어

| 용어 | 설명 |
|---|---|
| Aspect | 공통 기능 클래스 |
| Advice | 실제 실행될 공통 기능 |
| Target | 공통 기능이 적용될 대상 객체 |
| Proxy | Target 앞에서 요청을 가로채는 객체 |
| JoinPoint | Advice가 적용 가능한 실행 지점 |
| Weaving | Target에 Advice를 적용하는 과정 |

## @Before

대상 메서드 실행 전에 동작한다.

```java
@Before("LoggingAspect.logPointcut()")
public void logBefore(JoinPoint joinPoint) {

    System.out.println("Before Target : " + joinPoint.getTarget());

    System.out.println("Before Signature : "
            + joinPoint.getSignature());

    if(joinPoint.getArgs().length > 0) {
        System.out.println(joinPoint.getArgs()[0]);
    }
}
```

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| getTarget() | 실제 대상 객체 |
| getSignature() | 메서드 정보 |
| getArgs() | 전달된 인자값 |



### 실행 흐름

```text
메서드 실행 전
→ logBefore()
→ 실제 메서드 실행
```

---

## @After

대상 메서드 실행 후 무조건 실행된다.

```java
@After("logPointcut()")
public void logAfter(JoinPoint joinPoint) {

    System.out.println("After : "
            + joinPoint.getTarget());

    System.out.println("After : "
            + joinPoint.getSignature());
}
```


### 특징

예외 발생 여부와 상관없이 실행된다.

`try-finally` 의 `finally` 느낌과 비슷하다.

---

## @AfterReturning

메서드가 정상 종료되었을 때 실행된다.

```java
@AfterReturning(
        pointcut = "logPointcut()",
        returning = "result"
)
public void logAfterReturning(
        JoinPoint joinPoint,
        Object result
) {

    System.out.println("result : " + result);
}
```


### returning 속성

```java
returning = "result"
```

는

```java
Object result
```

매개변수 이름과 반드시 동일해야 한다.


### 특징

정상적으로 return 된 결과값을 받을 수 있다.



### 반환값 가공

```java
if(result instanceof Map) {

    ((Map<Long, MemberDTO>) result)
            .put(100L, new MemberDTO(100, "반환 값 가공"));
}
```

리턴 데이터를 수정할 수도 있다.

---

## @AfterThrowing

예외 발생 시 실행된다.

```java
@AfterThrowing(
        pointcut = "logPointcut()",
        throwing = "exception"
)
public void logAfterThrowing(Throwable exception) {

    System.out.println(exception);
}
```


### 특징

- 예외 발생 시에만 동작
- 예외 로그 처리에 많이 사용


### throwing 속성

```java
throwing = "exception"
```

은 매개변수 이름과 동일해야 한다.

## @Around

대상 메서드를 감싸서 실행한다.

가장 강력한 Advice이다.

```java
@Around("logPointcut()")
public Object logAround(
        ProceedingJoinPoint joinPoint
) throws Throwable {

    System.out.println("Around Before");

    Object result = joinPoint.proceed();

    System.out.println("Around After");

    return result;
}
```

---

## ProceedingJoinPoint

실제 대상 메서드를 실행할 수 있는 객체이다.


## proceed()

```java
joinPoint.proceed();
```

를 호출해야 실제 대상 메서드가 실행된다.

호출하지 않으면 메서드 자체가 실행되지 않는다.

---

## 실행 흐름

```text
Around Before
→ Before
→ 실제 메서드 실행
→ AfterReturning
→ After
→ Around After
```

---

## @Around 활용 예시

### 실행 시간 측정

```java
long start = System.currentTimeMillis();

Object result = joinPoint.proceed();

long end = System.currentTimeMillis();

System.out.println(end - start);
```


### 권한 체크

```java
if(!isLogin()) {
    return null;
}
```

### 트랜잭션 처리

```text
시작
→ 메서드 실행
→ 성공 시 commit
→ 실패 시 rollback
```

## 전체 흐름 정리

| Advice | 실행 시점 |
|---|---|
| @Before | 메서드 실행 전 |
| @After | 메서드 실행 후 |
| @AfterReturning | 정상 반환 후 |
| @AfterThrowing | 예외 발생 시 |
| @Around | 메서드 전체 감싸기 |
