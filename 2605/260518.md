# JDBC: JDBC 핵심 개념 정리

## 1. JDBC(Java Database Connectivity)

### 1.1 JDBC 개념
JDBC는 Java에서 데이터베이스와 연결하여 SQL을 실행하기 위한 기술이다.  
Java 프로그램과 Database 사이를 연결하는 역할을 수행한다.

주요 구성 요소
- DriverManager
- Connection
- Statement
- PreparedStatement
- ResultSet

---

## 2. DriverManager

### 2.1 DriverManager 역할
데이터베이스와 연결(Connection 객체 생성)을 담당하는 클래스이다.

JDBC Driver를 메모리에 로딩한 뒤 Database 연결을 수행한다.

---

### 2.2 Driver 로딩

```java
Class.forName("com.mysql.cj.jdbc.Driver");
```

#### 특징
- JDBC Driver를 메모리에 등록
- 동적 로딩 방식 사용
- 반드시 예외 처리 필요

---

### 2.3 Connection 생성

```java
Connection con =
DriverManager.getConnection(url, user, password);
```

#### 주요 정보
- url : DB 주소
- user : 계정명
- password : 비밀번호

---

### 2.4 Properties 파일 사용

DB 연결 정보를 별도 파일로 관리하는 방식이다.

#### 장점
- 중복 코드 감소
- 유지보수 편리
- 설정 변경 용이
- 오타 가능성 감소

---

## 3. Connection

### 3.1 Connection 역할
Database와 실제 연결된 객체이다.

SQL 실행을 위한 Statement, PreparedStatement 객체를 생성할 수 있다.

---

### 3.2 주요 특징
- DB 연결 성공 시 생성
- 사용 후 반드시 close()
- 일반적으로 Template 클래스로 공통 관리

---

### 3.3 Template 패턴 활용

```java
public static void close(Connection con)
```

공통 close 메소드를 만들어 자원 반납을 관리한다.

#### 장점
- 코드 중복 감소
- 자원 관리 일관성 유지
- 유지보수성 향상

---

## 4. Statement

### 4.1 Statement 역할
SQL문을 실행하고 결과를 반환하는 객체이다.

---

### 4.2 사용 방법

```java
stmt = conn.createStatement();
rset = stmt.executeQuery(query);
```

#### 특징
- SQL문을 문자열로 직접 전달
- 실행 시마다 SQL 새로 컴파일
- 동적 SQL 처리 가능

---

## 5. PreparedStatement

### 5.1 PreparedStatement 개념
Statement를 상속한 객체로, 위치홀더(?)를 사용하는 SQL 실행 방식이다.

---

### 5.2 위치홀더 사용

```java
String query =
"INSERT INTO MEMBER VALUES(?, ?)";
```

```java
pstmt.setString(1, id);
pstmt.setString(2, password);
```

---

### 5.3 PreparedStatement 장점

#### 1) 수행 속도 향상
- SQL을 미리 컴파일 후 재사용
- 반복 실행 시 성능 우수

---

#### 2) SQL Injection 방어
사용자 입력값을 안전하게 처리한다.

악의적인 SQL 삽입 공격을 예방할 수 있다.

---

### 5.4 Statement와 차이점

| 구분 | Statement | PreparedStatement |
| --- | --- | --- |
| SQL 처리 | 문자열 직접 전달 | 위치홀더 사용 |
| 컴파일 | 매번 수행 | 최초 1회 |
| 성능 | 상대적으로 느림 | 상대적으로 빠름 |
| 보안 | 취약 | 안전 |

---

## 6. SQL Injection

### 6.1 SQL Injection 개념
악의적인 SQL문을 삽입하여 Database를 비정상적으로 조작하는 공격 방식이다.

PreparedStatement를 사용하면 대부분 방어 가능하다.

---

## 7. SQL XML 관리

### 7.1 XML 파일 분리
SQL문을 XML 파일로 분리하여 관리할 수 있다.

#### 장점
- SQL 관리 편리
- 유지보수 향상
- 코드와 SQL 분리 가능

---

## 8. ResultSet

### 8.1 ResultSet 역할
SELECT 결과를 저장하는 객체이다.

조회 결과 테이블을 관리한다.

---

### 8.2 주요 메소드

#### next()

```java
rset.next()
```

다음 행 존재 여부 확인

- 존재 : true
- 없음 : false

---

#### getString()

```java
rset.getString("EMP_NAME")
```

현재 행의 컬럼 데이터를 조회한다.

자료형에 따라
- getInt()
- getDouble()
- getDate()

등을 사용할 수 있다.

---

### 8.3 반복 조회

```java
while(rset.next()) {
    System.out.println(
        rset.getString("EMP_ID")
    );
}
```

조회 결과를 한 행씩 반복 처리한다.

---

## 9. JDBC 자원 반납

### 9.1 close() 중요성
JDBC 객체는 사용 후 반드시 자원을 반납해야 한다.

반납 대상
- ResultSet
- Statement
- Connection

---

### 9.2 자원 반납 순서

일반적으로 역순으로 반납한다.

1. ResultSet
2. Statement
3. Connection
