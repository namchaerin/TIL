# Servlet 기반 Memo API 개발 흐름

## 프로젝트 구성
- Frontend : Next.js
- Backend : Servlet + Tomcat(WAS)
- Database : MySQL

---

## 메모 등록 API 흐름 (Create)

### 1. 클라이언트 요청
- 사용자가 메모 작성 후 등록 버튼 클릭
- Next.js에서 POST 요청 전송

```json
{
  "content": "오늘 학습 내용 정리"
}
```

---

### 2. Servlet 요청 처리
- Tomcat이 요청 URL을 확인하여 Servlet 실행
- `doPost()` 메소드 호출
- HttpServletRequest, HttpServletResponse 객체 생성

---

### 3. Service 계층 처리
- 입력값 검증
- 비즈니스 로직 수행
- DAO 호출

---

### 4. DAO + JDBC 처리
- Connection 생성
- PreparedStatement 생성
- INSERT SQL 실행

```sql
INSERT INTO tbl_memo(content) VALUES (?);
```

---

### 5. MySQL 저장
- 메모 데이터 저장
- memo_id 자동 생성
- created_at 저장

---

### 6. 응답 반환
- 등록 내용 반환
- JSON 형태로 응답

```json
{
    "content":"오늘 학습 내용 정리",
    "createdAt":"2026-05-21 16:23:16",
    "id":4
}
```

---

## 메모 조회 API 흐름 (Read)

### 1. 클라이언트 요청
- 페이지 로딩 시 GET 요청 전송

```http
GET /api/memos
```

---

### 2. Servlet 실행
- Tomcat이 요청 전달
- `doGet()` 메소드 실행

---

### 3. Service → DAO 호출
- 조회 로직 수행
- DAO에서 DB 조회 진행

---

### 4. JDBC 조회 처리

```sql
SELECT memo_id, content, created_at
FROM tbl_memo
ORDER BY memo_id DESC;
```

- ResultSet 반환
- DTO 객체로 데이터 변환

---

### 5. JSON 응답 반환

```json
[
    {
        "content": "안녕하세요~",
        "createdAt": "2026-05-21 14:41:12",
        "id": 3
    },
    {
        "content": "Next.js 미니 프로젝트에서는 fetch 주소만 맞추기",
        "createdAt": "2026-05-21 14:36:42",
        "id": 2
    },
    {
        "content": "Servlet API에 JDBC 붙이기",
        "createdAt": "2026-05-21 14:36:42",
        "id": 1
    }
]
```

---

### 6. Next.js 화면 렌더링
- API 응답 데이터 수신
- state 저장
- 메모 목록 화면 출력

---

# 핵심 개념 정리

## Servlet
- Controller 역할 수행
- 요청(Request) / 응답(Response) 처리

## Service
- 비즈니스 로직 처리
- 트랜잭션 단위 관리

## DAO
- 데이터베이스 접근 전담
- SQL 실행 담당

## JDBC
- Java와 DB 연결
- SQL 실행 및 결과 반환 처리

## DTO
- 계층 간 데이터 전달 객체
- DB 조회 결과 저장

---

# 전체 흐름 요약

```text
Next.js
   ↓
Servlet (Controller)
   ↓
Service
   ↓
DAO (JDBC)
   ↓
MySQL
   ↓
DTO 변환
   ↓
JSON 응답
   ↓
Next.js 렌더링
```
