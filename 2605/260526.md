# 1. Spring DI(Dependency Injection)

## 1-1. DI(의존성 주입)란

### DI 개념

- DI(Dependency Injection)는 객체가 직접 다른 객체를 생성하지 않고, 스프링 컨테이너가 객체를 생성하여 주입해주는 방식이다.
- 객체 간의 결합도를 낮추고 유지보수성과 확장성을 높일 수 있다.

```java
// 강한 결합
private BookDAO bookDAO = new BookDAOImpl();
```

- 위 방식은 구현체가 변경되면 직접 수정해야 하는 문제가 있다.

```java
// 느슨한 결합
@Autowired
private BookDAO bookDAO;
```

- 스프링 컨테이너가 자동으로 Bean을 찾아 주입해준다.

---

## 1-2. Spring Bean

### Bean이란

- 스프링 컨테이너가 생성하고 관리하는 객체를 Bean이라고 한다.
- `@Component` 계열 어노테이션을 통해 Bean으로 등록한다.

| 어노테이션 | 역할 |
| --- | --- |
| `@Component` | 일반 Bean 등록 |
| `@Repository` | DAO 계층 |
| `@Service` | Service 계층 |
| `@Controller` | Controller 계층 |

---

## 1-3. @Autowired

### @Autowired 개념

- `@Autowired`는 타입(Type)을 기준으로 Bean을 자동 주입하는 어노테이션이다.
- 스프링 컨테이너가 동일 타입의 Bean을 찾아 의존성을 주입한다.

---

# 2. DI Annotation

## 2-1. 필드(Field) 주입

### 2-1-1. 필드 주입 방식

- 필드에 직접 `@Autowired`를 작성하여 의존성을 주입하는 방식이다.

```java
@Service
public class BookService {

    @Autowired
    private BookDAO bookDAO;
}
```

### 2-1-2. 특징

- 코드가 간단하다.
- 테스트가 어렵다.
- 객체 생성 후 의존성이 주입된다.
- final 사용이 불가능하다.

---

## 2-2. 생성자(Constructor) 주입

### 2-2-1. 생성자 주입 방식

- 생성자를 통해 의존성을 주입하는 방식이다.

```java
@Service
public class BookService {

    private final BookDAO bookDAO;

    @Autowired
    public BookService(BookDAO bookDAO) {
        this.bookDAO = bookDAO;
    }
}
```

### 2-2-2. 특징

- 객체 생성 시점에 의존성이 함께 주입된다.
- final 사용 가능 → 객체 불변성 보장
- 테스트 코드 작성이 쉽다.
- 의존성이 없으면 객체 생성 자체가 실패한다.
- 가장 권장되는 방식이다.

### 2-2-3. Spring 4.3 이후 특징

- 생성자가 1개라면 `@Autowired` 생략 가능하다.

```java
public BookService(BookDAO bookDAO) {
    this.bookDAO = bookDAO;
}
```

---

## 2-3. Setter 주입

### 2-3-1. Setter 주입 방식

- Setter 메소드에 `@Autowired`를 작성하여 주입한다.

```java
@Service
public class BookService {

    private BookDAO bookDAO;

    @Autowired
    public void setBookDAO(BookDAO bookDAO) {
        this.bookDAO = bookDAO;
    }
}
```

### 2-3-2. 특징

- 선택적인 의존성 주입에 사용 가능하다.
- 객체 생성 후 의존성을 변경할 수 있다.
- 런타임 시점에 Null 오류가 발생할 가능성이 있다.

# 3. 스프링 컨테이너

## 3-1. 스프링 컨테이너 역할

### 주요 기능

- 객체 생성
- Bean 관리
- 의존성 주입(DI)
- 생명주기 관리

```java
ApplicationContext context =
    new AnnotationConfigApplicationContext("패키지명");
```

- 스프링 컨테이너가 패키지를 스캔하여 Bean을 등록한다.

---

## 3-2. Bean 조회

### Bean 가져오기

```java
BookService bookService =
    context.getBean("bookServiceConstructor", BookService.class);
```

- 등록된 Bean 객체를 가져와 사용할 수 있다.
