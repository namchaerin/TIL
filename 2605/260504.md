# Java: 클래스와 객체
## 1. 추상화
- 추상화는 객체가 수행해야 할 핵심적인 역할과 기능만을 외부에 드러내고, 내부 구현은 숨기는 개념이다.
- 객체 간의 상호작용은 내부 구현이 아닌 "행동(메서드)"을 통해 이루어지며, 이를 통해 객체의 책임이 명확하게 분리된다.
- 이러한 구조는 내부 구현이 변경되더라도 외부 코드에 영향을 주지 않기 때문에 유지보수성과 확장성을 높이는 데 유리하다.

#### Car 클래스
```java
public class Car {

    private boolean isOn;   // 시동 상태
    private int speed;      // 현재 시속

    /* 자동차가 직접 할 수 있는 행동(메서드) 정의 */
    public void startUp(){ ... }

    public void go(){ ... }

    public void stop(){ ... }

    public void turnOff(){ ... }
}
```
- Car 클래스에서는 시동, 가속, 정지, 종료와 같은 자동차의 핵심 기능만 메서드로 정의하고, 실제 동작 방식은 내부에 감추고 있다.


#### CarRacer 클래스

```java
public class CarRacer {

    // 자신의 자동차를 가지고 있다. (소유 관계 / has-a  관계)
    private final Car myCar = new Car();

    public void starUp() {
        // 카레이서는 시동 거는 방법을 직접 모른다.
        // 단지 자신의 차에게 '시동 걸어!' 라고 요청(메세지 전송) 할 뿐이다.
        myCar.startUp();
    }

    // 엑셀 밟도록 명령
    public void stepAccelator() {
        myCar.go();
    }

    public void stepBreak() {
        myCar.stop();
    }

    public void turnOff() {
        myCar.turnOff();
    }
}
```
- CarRacer 클래스는 자동차의 내부 동작 원리를 알 필요 없이, 단순히 "시동을 걸어라", "달려라"와 같은 메시지를 전달하여 기능을 사용한다.


#### Application 클래스
```java
public static void main(String[] args) {

    CarRacer racer = new CarRacer();
    Scanner sc = new Scanner(System.in);

    while (true) {
        printMenu();
        int no = sc.nextInt();

        switch (no) {
            case 1 -> racer.starUp();
            case 2 -> racer.stepAccelator();
            case 3 -> racer.stepBreak();
            case 4 -> racer.turnOff();
            case 9 -> {
                System.out.println("프로그램 종료");
                return;
            }
            default -> System.out.println("잘못된 입력");
        }
    }
}
```
- 사용자 → CarRacer → Car 순으로 요청이 전달되는 구조
- 객체 간 메시지 전달을 통해 기능이 수행됨
- 내부 구현이 아닌 기능 단위로 프로그램을 제어함

## 2. 생성자와 객체 초기화

- 생성자는 객체가 생성될 때 자동으로 호출되는 초기화 메소드이다.
- 객체의 필드를 원하는 값으로 설정하는 역할을 하며, 클래스명과 동일하고 반환 타입을 가지지 않는다.

#### 생성자 종류

- 기본 생성자: 매개변수가 없으며 객체의 기본 상태를 초기화한다.
- 매개변수 생성자: 외부에서 값을 전달받아 객체 생성 시점에 필드를 초기화한다.


### this와 생성자 재사용

- this는 현재 생성된 객체를 가리키며, 필드와 매개변수를 구분하는 데 사용된다.
- this()는 같은 클래스의 다른 생성자를 호출할 때 사용되며, 반드시 생성자의 첫 줄에 작성해야 한다.
- 이를 통해 공통 초기화 로직을 재사용하고 코드 중복을 줄일 수 있다.

👉 예: id, pwd, name 초기화 후 추가적으로 enrollDate 설정


### 캡슐화

- 필드를 private으로 선언하여 외부 접근을 제한한다.
- getter/setter를 통해서만 데이터를 조회 및 수정할 수 있다.
- 이를 통해 객체의 상태를 안전하게 관리할 수 있다.

## 3. 메서드 오버로딩

- 오버로딩은 같은 이름의 메서드를 여러 개 정의하는 기술이다.
- 단, 메서드 시그니처가 달라야 한다.

### 성립 조건

- 매개변수 개수가 다른 경우
- 매개변수 타입이 다른 경우
- 매개변수 순서가 다른 경우


### 불성립 조건

- 반환 타입만 다른 경우 → 불가능
- 접근 제한자만 다른 경우 → 불가능
- 매개변수 이름만 다른 경우 → 불가능

👉 즉, 컴파일러는 메서드 시그니처로만 메서드를 구분한다.
