# Java: static, final, 변수 종류 및 객체 배열

## 1. static 키워드
static은 **클래스**가 로딩되는 시점에 메모리의 정적 영역에 할당되는 키워드이다.  
static으로 선언된 필드나 메소드는 객체 생성 없이 `클래스명.필드` 또는 `클래스명.메소드()` 형태로 접근할 수 있다.  

또한 static은 여러 인스턴스가 값을 공유해야 하는 경우에 사용된다. 따라서 공통적으로 사용하는 데이터(예: 카운트 값, 설정 정보 등)를 관리할 때 적합하다.

---

## 2. final 키워드
final은 변경 불가의 의미를 가지는 키워드이다.  

- 변수에 사용하면 값 재할당이 불가능하다.
- 메소드에 사용하면 오버라이딩이 불가능하다.
- 클래스에 사용하면 상속이 불가능하다.

특히 final 변수는 반드시 초기화가 필요하다.  
초기화 방법은 다음 두 가지이다.
- 선언과 동시에 초기화
- 생성자를 통한 초기화

---

## 3. 변수의 종류
클래스에서 사용하는 변수는 크게 세 가지로 나뉜다.

### 3.1 클래스 변수 (static 변수)
- static 키워드를 사용하여 선언
- 생성 시점: 프로그램 시작 시
- 소멸 시점: 프로그램 종료 시
- 모든 인스턴스가 값을 공유

### 3.2 인스턴스 변수
- 클래스의 필드에 static 없이 선언
- 생성 시점: 객체 생성 시
- 소멸 시점: 더 이상 참조되지 않을 때 (GC 대상)
- 객체마다 별도의 값을 가짐

### 3.3 지역 변수
- 메소드, 생성자 내부에서 선언
- 생성 시점: 메소드 호출 시
- 소멸 시점: 메소드 종료 시
- 가장 짧은 생명주기를 가짐

변수는 선언 위치에 따라 생성 시점과 소멸 시점, 그리고 사용 범위가 달라진다.

---

## 4. 초기화 블럭
초기화 블럭은 객체 생성 시 공통적으로 실행되는 코드를 작성하는 영역이다.  
생성자와 별도로 공통 초기화 로직을 구성할 수 있다.

### 4.1 인스턴스 초기화 블럭
- 객체 생성 시마다 실행된다.

### 4.2 static 초기화 블럭
- 클래스가 로딩될 때 한 번만 실행된다.

초기화 블럭을 사용하면 여러 생성자에서 중복되는 초기화 코드를 줄일 수 있다.

---

## 5. 객체 배열
객체 배열은 동일한 클래스 타입의 객체들을 배열 형태로 관리하는 구조이다.  
배열에는 객체 자체가 아닌 객체의 참조값이 저장된다.

### 5.1 특징
- 동일한 타입의 객체를 한 번에 관리할 수 있다.
- 반복문을 통해 객체의 메소드를 효율적으로 호출할 수 있다.

### 5.2 사용 흐름
1. 객체 배열 생성
2. 각 인덱스에 객체 생성 후 할당
3. 반복문을 통해 객체의 기능 실행

### 5.3 예시 코드
```java
Car[] carArray = new Car[5];
carArray[0] = new Car("페라리", 300);
carArray[1] = new Car("람보르기니", 350);
carArray[2] = new Car("롤스로이스", 250);
carArray[3] = new Car("부가티베이론", 400);
carArray[4] = new Car("포터", 500);

for (int i = 0; i < carArray.length; i++) {
    carArray[i].driveMaxSpeed();
}
```
