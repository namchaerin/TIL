# Cookie & Session

## 1. Cookie

### 1-1. Cookie 개념
- 클라이언트(브라우저)에 저장되는 데이터
- key-value 형태로 저장
- 서버 요청 시 자동 전달

### 1-2. Cookie 특징
- 클라이언트 저장 방식
- 보안 취약 (로컬 저장)
- 크기/개수 제한 존재

### 1-3. Cookie 주요 속성
- name / value : 기본 데이터
- maxAge : 유지 시간
- path : 적용 경로
- domain : 적용 도메인
- secure : HTTPS에서만 전송

### 1-4. Cookie 생성 및 전송
```java
Cookie cookie = new Cookie("name", "value");
cookie.setMaxAge(60 * 60);
response.addCookie(cookie);
```

### 1-5. Cookie 조회
```java
Cookie[] cookies = request.getCookies();

for (Cookie c : cookies) {
    System.out.println(c.getName());
    System.out.println(c.getValue());
}
```

## 2. Session

### 2-1. Session 개념
- 서버에 저장되는 사용자 상태 정보
- 클라이언트는 Session ID만 보유
- 보안성이 Cookie보다 높음

---

### 2-2. Session 생성
```java
HttpSession session = request.getSession();
```

---

### 2-3. Session 데이터 저장 및 조회
```java
session.setAttribute("key", value);

Object value = session.getAttribute("key");
```

---

### 2-4. Session 유지 시간 설정
```java
session.setMaxInactiveInterval(60 * 10);
```

---

### 2-5. Session 삭제
```java
session.removeAttribute("key");
session.invalidate();
```

---

### 2-6. Session 주요 메서드

| 메서드 | 설명 |
|------|------|
| setAttribute | 데이터 저장 |
| getAttribute | 데이터 조회 |
| removeAttribute | 데이터 삭제 |
| getId | 세션 ID 반환 |
| isNew | 신규 세션 여부 |
| invalidate | 세션 전체 삭제 |

---

## Cookie vs Session 비교

| 구분 | Cookie | Session |
|------|--------|--------|
| 저장 위치 | 클라이언트 | 서버 |
| 보안 | 낮음 | 높음 |
| 용량 | 제한 있음 | 제한 거의 없음 |
| 생명주기 | 설정 가능 | 기본 30분 |
