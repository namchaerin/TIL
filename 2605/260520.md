# Servlet 및 Web/WAS 개요

## 1. 네트워크 통신 개요

### 1-1. Server - Client 구조

- Server - 서비스를 제공하는 컴퓨터
- Client - 서버의 서비스를 사용하는 사용자

클라이언트가 요청(Request)을 보내면 서버가 응답(Response)을 반환하는 구조이다.

---

## 2. Web Server

### 2-1. Web Server 개념

정적인 페이지(HTML, CSS, 이미지 파일 등)를 사용자에게 제공하는 서버이다.

특징
- 빠른 처리 속도
- 정적 페이지 처리
- 단순한 요청 처리

대표 종류
- Apache
- IIS
- Nginx

---

## 3. WAS(Web Application Server)

### 3-1. WAS 개념

동적인 웹 페이지를 생성하여 사용자에게 제공하는 서버이다.

특징
- 비즈니스 로직 처리
- 데이터베이스 연동 가능
- 동적 페이지 생성
- Servlet/JSP 실행 가능

대표 종류
- Apache Tomcat
- JBoss
- Jeus

---

## 4. Web Server와 WAS 차이

| 구분 | Web Server | WAS |
| --- | --- | --- |
| 역할 | 정적 데이터 제공 | 동적 데이터 처리 |
| 처리 방식 | HTML 파일 전달 | Servlet/JSP 실행 |
| 속도 | 빠름 | 상대적으로 느림 |
| 특징 | 단순 응답 처리 | 비즈니스 로직 처리 |

---

## 5. CGI와 WAS

### 5-1. CGI 방식

요청마다 새로운 프로그램을 실행하는 방식이다.

단점
- 요청 수 증가 시 성능 저하
- 자원 사용량 증가

---

### 5-2. WAS 방식

하나의 프로그램이 여러 요청을 처리하는 방식이다.

장점
- 효율적인 자원 사용
- 멀티스레드 처리 가능
- 성능 향상

---

## 6. Servlet Container

### 6-1. Servlet Container 역할

Servlet의 실행과 생명주기를 관리하는 객체이다.

주요 기능
- Servlet 생성 및 소멸 관리
- 요청/응답 객체 생성
- 멀티스레드 처리
- URL 요청 매핑

대표 예시
- Tomcat

---

## 7. JSP Container

### 7-1. JSP Container 역할

JSP 파일을 Servlet 형태로 변환하여 실행하는 역할을 한다.

처리 과정

```text
JSP → Servlet 변환 → 컴파일 → 실행
```

---

## 8. Servlet Lifecycle

### 8-1. init()

Servlet 최초 생성 시 한 번만 실행된다.

특징
- 초기 설정 수행
- DB 연결 준비
- 자원 초기화

```java
public void init() {}
```

---

### 8-2. service()

클라이언트 요청이 들어올 때마다 실행된다.

특징
- GET/POST 요청 처리
- 비즈니스 로직 수행
- 가장 많이 호출되는 메소드

```java
protected void service() {}
```

---

### 8-3. destroy()

서버 종료 또는 Servlet 제거 시 한 번 실행된다.

특징
- 자원 반납
- 연결 종료
- 메모리 정리

```java
public void destroy() {}
```

---

## 9. Servlet Lifecycle 흐름

```text
클라이언트 요청
        ↓
Servlet 생성
        ↓
init() 실행 (1회)
        ↓
service() 실행 (요청마다 반복)
        ↓
destroy() 실행 (종료 시 1회)
```
