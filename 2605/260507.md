# Java: 상속, 다형성, 추상 클래스와 인터페이스

## 1. 상속(Inheritance)

### 1.1 상속의 개념
상속은 부모 클래스의 필드와 메소드를 자식 클래스가 물려받아 자신의 것처럼 사용하는 기술이다.  
자식 클래스는 `extends` 키워드를 사용하여 부모 클래스를 상속받는다.

```java
public class Academy extends Company {

}
```

자바는 단일 상속만 지원하며, 생성자는 상속되지 않는다.

### 1.2 상속을 사용하는 이유
- 기존 클래스의 기능 재사용 가능
- 클래스 간 계층 구조 형성 가능
- 다형성 구현의 기반 제공
- 코드 중복 감소 및 유지보수 향상

---

## 2. 상속 관련 키워드

### 2.1 IS-A 관계
IS-A 관계는 “자식 클래스는 부모 클래스이다”라는 의미를 가진다.

예시:
- Circle is a Shape
- 원은 하나의 도형이다.

상속 관계가 적절한지 판단할 때 사용하는 기준이다.

---

### 2.2 super와 super()

#### super
부모 클래스의 멤버에 접근하기 위한 참조 변수이다.

```java
@Override
public String getInformation() {

    return super.getInformation()
            + "Computer ["
            + "cpu=" + this.cpu
            + "]";
}
```

#### super()
부모 클래스의 생성자를 호출하는 구문이다.

```java
public Computer(String cpu) {

    super();

    this.cpu = cpu;
}
```

---

### 2.3 오버라이딩(Overriding)
부모 클래스의 메소드를 자식 클래스에서 재정의하는 기술이다.

#### 오버라이딩 조건
- 메소드명 동일
- 리턴타입 동일
- 매개변수 동일
- 접근제어자는 같거나 더 넓어야 함
- private, final 메소드는 오버라이딩 불가능

---

### 2.4 오버라이딩과 오버로딩 차이
- 오버라이딩: 상속 관계에서 메소드 재정의
- 오버로딩: 같은 이름의 메소드를 여러 개 정의

---

# 2. 다형성(Polymorphism)

## 2.1 다형성의 개념
하나의 객체가 여러 타입으로 사용될 수 있는 특징이다.

부모 타입 하나로 여러 자식 객체를 관리할 수 있으며, 같은 메소드 호출이라도 객체마다 다르게 동작할 수 있다.

---

## 2.2 다형성의 장점
- 유지보수성 향상
- 코드 재사용 증가
- 확장성 향상
- 결합도 감소

---

## 2.3 동적 바인딩
컴파일 시점에는 부모 타입 메소드와 연결되지만, 실행 시점에는 실제 객체의 오버라이딩된 메소드가 호출되는 방식이다.

즉, 실행 시 실제 객체 타입에 맞는 메소드가 동작한다.

---

## 2.4 업캐스팅과 다운캐스팅

### 업캐스팅
자식 객체를 부모 타입으로 형변환하는 것  
자동 형변환이 이루어진다.

### 다운캐스팅
부모 타입을 자식 타입으로 형변환하는 것  
명시적으로 작성해야 한다.

---

## 2.5 instanceof 연산자
객체의 실제 타입을 확인하는 연산자이다.  
형변환 전에 타입을 확인하여 안전하게 사용할 수 있다.

```java
if(car instanceof Sonata) {
    ((Sonata) car).moveSonata();
}
```

---

# 3. 추상 클래스(Abstract Class)

## 3.1 추상 클래스 개념
추상 메소드를 0개 이상 포함하는 클래스이다.  
직접 객체 생성이 불가능하며, 상속을 통해 사용한다.

추상 메소드는 선언만 존재하고 기능 구현은 없는 메소드이다.

---

## 3.2 추상 클래스를 사용하는 이유
- 공통 기능 제공
- 다형성 구현 가능
- 자식 클래스에 메소드 구현 강제 가능
- 일관된 구조 유지 가능

---

# 4. 인터페이스(Interface)

## 4.1 인터페이스 개념
추상 메소드와 상수만 가질 수 있는 구조이다.  
`implements` 키워드를 사용하여 구현한다.

```java
interface TestInter {

    double PI = 3.14;

    void method();
}
```

```java
class Person implements TestInter {

    @Override
    public void method() {

    }
}
```

---

## 4.2 인터페이스를 사용하는 이유
- 공통 기능 강제
- 다중 구현 가능
- 결합도 감소
- 유지보수성과 확장성 향상

---

# 5. 추상 클래스와 인터페이스 비교

| 구분 | 추상 클래스 | 인터페이스 |
| --- | --- | --- |
| 사용 키워드 | extends | implements |
| 상속 방식 | 단일 상속 | 다중 구현 가능 |
| 메소드 형태 | 일반 메소드 + 추상 메소드 | 추상 메소드 중심 |
| 객체 생성 | 불가능 | 불가능 |
