# Java: Collection Framework

## 1. Collection Framework 개요

Collection Framework는 많은 데이터를 효율적으로 저장하고 관리하기 위한 클래스와 인터페이스의 집합이다.  
자바에서는 List, Set, Map 등의 자료구조를 제공하여 데이터 관리와 유지보수를 쉽게 할 수 있다.

### Collection을 사용하는 이유
- 일관된 API 제공
- 자료구조 구현 부담 감소
- 데이터 처리 속도 및 유지보수성 향상

---

## 2. List

### 2.1 List 특징
- 저장 순서 유지
- 중복 데이터 허용
- 인덱스를 이용한 접근 가능

대표 클래스:
- ArrayList
- LinkedList
- Stack

---

### 2.2 ArrayList

#### 특징
- 내부적으로 배열 사용
- 조회 성능 우수
- 데이터 추가/삭제 시 배열 복사 발생 가능

#### 데이터 추가
```java
List<String> list = new ArrayList<>();

list.add("java");
list.add("spring");
```

#### 데이터 삭제
```java
list.remove("java");
list.remove(0);
```

---

### 2.3 LinkedList

#### 특징
- 링크 구조로 데이터 연결
- 삽입/삭제 성능 우수
- 조회 성능은 ArrayList보다 느림

#### 데이터 추가
```java
LinkedList<String> list = new LinkedList<>();

list.add("mysql");
list.addFirst("java");
list.addLast("spring");
```

#### 데이터 삭제
```java
list.remove();
list.removeFirst();
list.removeLast();
```

---

### 2.4 Stack

#### 특징
- LIFO 구조 (Last In First Out)
- 나중에 들어간 데이터가 먼저 나옴

#### 데이터 추가
```java
Stack<Integer> stack = new Stack<>();

stack.push(10);
stack.push(20);
```

#### 데이터 삭제
```java
stack.pop();
```

---

### 2.5 Queue

#### 특징
- FIFO 구조 (First In First Out)
- 먼저 들어간 데이터가 먼저 나옴

#### 데이터 추가
```java
Queue<String> queue = new LinkedList<>();

queue.offer("A");
queue.offer("B");
```

#### 데이터 삭제
```java
queue.poll();
```

---

## 3. Set

### 3.1 Set 특징
- 중복 데이터 저장 불가
- 저장 순서 보장하지 않음
- null은 하나만 저장 가능

대표 클래스:
- HashSet
- LinkedHashSet
- TreeSet

---

### 3.2 HashSet

#### 특징
- 해시 기반 저장 구조
- 중복 허용 안함
- 빠른 검색 가능

#### 데이터 추가
```java
Set<String> set = new HashSet<>();

set.add("java");
set.add("spring");
```

#### 데이터 삭제
```java
set.remove("java");
```

---

### 3.3 LinkedHashSet

#### 특징
- HashSet과 동일
- 저장 순서 유지

#### 데이터 추가
```java
Set<String> set = new LinkedHashSet<>();

set.add("A");
set.add("B");
```

#### 데이터 삭제
```java
set.remove("A");
```

---

### 3.4 TreeSet

#### 특징
- 자동 정렬 지원
- 중복 저장 불가
- 이진 검색 트리 구조 사용

#### 데이터 추가
```java
TreeSet<Integer> set = new TreeSet<>();

set.add(30);
set.add(10);
set.add(20);
```

#### 데이터 삭제
```java
set.remove(10);
```

---

## 4. Map

### 4.1 Map 특징
- Key와 Value 형태로 저장
- Key 중복 불가
- Value 중복 가능

대표 클래스:
- HashMap
- TreeMap
- Properties

---

### 4.2 HashMap

#### 특징
- 가장 많이 사용하는 Map 구현체
- 검색 성능 우수
- Key 기반 데이터 관리

#### 데이터 추가
```java
Map<String, String> map = new HashMap<>();

map.put("id", "user01");
map.put("name", "홍길동");
```

#### 데이터 삭제
```java
map.remove("id");
```

---

### 4.3 TreeMap

#### 특징
- Key 기준 자동 정렬
- 검색 및 범위 조회에 활용

#### 데이터 추가
```java
TreeMap<Integer, String> map = new TreeMap<>();

map.put(3, "C");
map.put(1, "A");
```

#### 데이터 삭제
```java
map.remove(1);
```

---

### 4.4 Properties

### 특징
- Key와 Value를 모두 String으로 저장
- 환경 설정 파일 관리에 주로 사용

### 데이터 추가
```java
Properties prop = new Properties();

prop.setProperty("url", "localhost");
prop.setProperty("driver", "mysql");
```

### 데이터 삭제
```java
prop.remove("url");
```
