# sql: Transaction, DDL, 제약조건(CONSTRAINT)

## 1. Transaction

### 1.1 Transaction 개념
Transaction은 데이터베이스 작업을 하나의 논리적인 작업 단위로 처리하는 기능이다.  
모든 작업이 성공하면 COMMIT되고, 하나라도 실패하면 ROLLBACK하여 이전 상태로 복구한다.

즉, “모두 성공하거나 모두 실패”하는 특징을 가진다.

---

### 1.2 주요 명령어

#### START TRANSACTION
트랜잭션 시작

```sql
START TRANSACTION;
```

---

#### COMMIT
작업 내용을 최종 저장

```sql
COMMIT;
```

---

#### ROLLBACK
트랜잭션 시작 이전 상태로 복구

```sql
ROLLBACK;
```

---

#### AUTOCOMMIT OFF
자동 커밋 비활성화

```sql
SET autocommit = OFF;
```

---

## 2. DDL(Data Definition Language)

### 2.1 DDL 개념
DDL은 데이터베이스 구조를 정의하거나 변경하는 명령어이다.

주요 명령어
- CREATE
- ALTER
- DROP

---

## 3. CREATE TABLE

테이블 생성 시 사용한다.

```sql
CREATE TABLE IF NOT EXISTS tb1 (
    pk INT PRIMARY KEY,
    fk INT,
    col1 VARCHAR(255)
);
```

---

### 3.1 AUTO_INCREMENT

PRIMARY KEY 값을 자동 증가시키는 기능이다.

```sql
pk INT AUTO_INCREMENT PRIMARY KEY
```

INSERT 시 번호를 자동 생성한다.

---

## 4. ALTER TABLE

기존 테이블 구조를 수정할 때 사용한다.

---

### 4.1 ADD

컬럼 추가

```sql
ALTER TABLE tb2
ADD col2 INT NOT NULL;
```

---

### 4.2 DROP COLUMN

컬럼 삭제

```sql
ALTER TABLE tb2
DROP COLUMN col2;
```

---

### 4.3 CHANGE COLUMN

컬럼명과 데이터 타입 변경

```sql
ALTER TABLE tb2
CHANGE COLUMN fk change_fk INT NOT NULL;
```

---

### 4.4 MODIFY

컬럼 속성 변경

```sql
ALTER TABLE tb2
MODIFY pk INT;
```

---

## 5. 제약조건(CONSTRAINT)

### 5.1 NOT NULL

NULL 값을 허용하지 않는 제약조건이다.

```sql
user_id VARCHAR(255) NOT NULL
```

필수 입력값 설정 시 사용한다.

---

### 5.2 UNIQUE

중복값을 허용하지 않는 제약조건이다.

```sql
user_id VARCHAR(255) UNIQUE
```

아이디, 이메일 등에 사용된다.

---

### 5.3 PRIMARY KEY

테이블의 대표 식별자 역할을 하는 제약조건이다.

특징
- NOT NULL
- UNIQUE

두 조건을 모두 가진다.

```sql
user_no INT PRIMARY KEY
```

---

### 5.4 FOREIGN KEY

다른 테이블의 값을 참조하는 제약조건이다.

테이블 간 관계를 연결할 때 사용한다.

```sql
FOREIGN KEY (grade_code)
REFERENCES user_grade (grade_code)
```

참조 테이블에 존재하는 값만 사용할 수 있다.

---

## 6. ON UPDATE / ON DELETE 옵션

### 6.1 SET NULL

부모 데이터 변경 또는 삭제 시 자식 데이터를 NULL로 변경한다.

```sql
ON UPDATE SET NULL
ON DELETE SET NULL
```

---

### 6.2 CASCADE

부모 데이터 변경 또는 삭제 시 자식 데이터도 함께 변경된다.

```sql
ON UPDATE CASCADE
ON DELETE CASCADE
```

---

## 7. CHECK

허용 가능한 데이터 조건을 제한하는 제약조건이다.

```sql
gender VARCHAR(3)
CHECK (gender IN ('남', '여'))
```

```sql
age INT CHECK(age >= 19)
```

데이터 유효성 검증에 사용된다.

---

## 8. DEFAULT

값을 입력하지 않았을 때 자동으로 기본값을 저장하는 기능이다.

```sql
user_name VARCHAR(255) DEFAULT '아무개'
```

```sql
add_day DATE DEFAULT(current_date)
```
