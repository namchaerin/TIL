# MyBatis SqlSessionFactory와 SqlSession

## MyBatis 개요

### MyBatis란

- JDBC를 보다 편리하게 사용할 수 있도록 도와주는 SQL 매퍼 프레임워크이다.
- SQL 작성과 데이터 매핑을 분리하여 개발 생산성을 높일 수 있다.
- JDBC의 반복적인 Connection, Statement, ResultSet 처리 코드를 줄일 수 있다.

### MyBatis 주요 구성 요소

| 구성 요소 | 역할 |
|-----------|------|
| SqlSessionFactory | SqlSession 생성 공장 |
| SqlSession | SQL 실행 및 트랜잭션 관리 |
| Mapper | SQL과 Java 메서드 연결 |
| DTO | 조회 결과를 담는 객체 |
| Config XML | DB 연결 정보 및 Mapper 등록 |

## SqlSessionFactory

### SqlSessionFactory의 역할

- MyBatis 설정 파일을 읽어 SqlSession을 생성하는 객체이다.
- 애플리케이션 실행 중 하나만 생성하여 사용하는 것이 일반적이다.
- 생성 비용이 크기 때문에 재사용한다.

### 싱글톤 패턴 적용

```java
private static SqlSessionFactory sqlSessionFactory;
```

- static 필드로 선언하여 애플리케이션 전체에서 하나의 객체만 사용한다.
- 클래스 로딩 시 메모리에 유지된다.
- 불필요한 객체 생성을 방지할 수 있다.

### 생성 과정

```java
String resource = "mybatis-config.xml";

InputStream inputStream =
        Resources.getResourceAsStream(resource);

sqlSessionFactory =
        new SqlSessionFactoryBuilder().build(inputStream);
```

동작 순서

1. mybatis-config.xml 파일 읽기
2. DB 연결 정보 로드
3. Mapper 정보 로드
4. SqlSessionFactory 생성
5. 이후 재사용

## SqlSession

### SqlSession의 역할

- 실제 SQL 실행을 담당한다.
- Mapper와 DB를 연결한다.
- 트랜잭션을 관리한다.
- CRUD 작업을 수행한다.

### 생성 과정

```java
SqlSession sqlSession =
        sqlSessionFactory.openSession(false);
```

- SqlSessionFactory를 통해 생성된다.
- 요청이 들어올 때마다 새로운 SqlSession이 생성된다.

### Auto Commit 설정

```java
openSession(false)
```

- Auto Commit 비활성화
- 개발자가 직접 commit(), rollback() 수행

```java
sqlSession.commit();
sqlSession.rollback();
```

## SqlSessionFactory와 SqlSession 생명주기 비교

| 구분 | SqlSessionFactory | SqlSession |
|--------|------------------|------------|
| 생성 시점 | 최초 1회 | 요청마다 생성 |
| 개수 | 1개 | 여러 개 |
| 역할 | SqlSession 생성 | SQL 실행 |
| 생명주기 | 애플리케이션 종료까지 유지 | 작업 종료 시 반환 |
| 패턴 | Singleton | Request Scope |
