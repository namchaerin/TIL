# 📓 개발 일지 (Day 1)

## 학습 주제
Git Rebase를 활용한 브랜치 정리 및 충돌 해결

## 학습 내용
- `feature/BE-recipe-comment` 브랜치를 `dev` 브랜치 기준으로 병합하기 위해 Rebase를 진행하였다.
- 충돌 원인을 분석한 결과, 레시피 상세 조회 기능(API-304)이 `dev`와 댓글 브랜치에 각각 다른 커밋으로 존재하여 중복 커밋으로 인한 충돌이 발생한 것을 확인하였다.
- `git rebase --onto dev ff1a584 feature/BE-recipe-comment` 명령을 사용하여 중복된 상세 조회 커밋은 제외하고 댓글 관련 커밋만 새로운 `dev` 기준으로 재배치하였다.
- 실제 브랜치에 적용하기 전 `git worktree`를 이용해 동일한 작업을 시뮬레이션하여 충돌 여부를 사전에 검증한 후 안전하게 적용하였다.

## 학습한 내용 요약
- `git rebase --onto`를 활용하면 특정 커밋을 제외하고 원하는 커밋만 새로운 브랜치 기준으로 재배치할 수 있다.
- `git worktree`를 이용하면 원본 브랜치를 변경하지 않고 Rebase 결과를 미리 검증할 수 있어 안전하게 작업할 수 있다.
