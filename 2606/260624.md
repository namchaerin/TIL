# Spring Security 인증/인가

## 1. JWT 기반 인증 정보 관리

JWT 인증 필터는 요청의 Access Token을 검증하고, 인증 성공 시 `Authentication` 객체를 생성해 `SecurityContext`에 저장한다. 이후 요청 처리 과정 전반에서 `SecurityContext`에 저장된 인증 정보를 꺼내 활용할 수 있다.

---

## 2. 로그인 사용자 정보 조회

### `@AuthenticationPrincipal`

Controller 메서드 파라미터에 `@AuthenticationPrincipal`을 붙이면 `SecurityContext`에서 현재 로그인한 사용자의 인증 객체를 바로 주입받을 수 있다.

### 기본 `UserDetails` vs `CustomUserDetails`

기본 `UserDetails`는 `username`과 권한 정보만 제공한다. `id`, `email` 같은 추가 필드에 접근하려면 `UserDetails`를 직접 구현한 `CustomUserDetails`가 필요하다.

```java
// 기본 UserDetails → username, 권한만 접근 가능
@AuthenticationPrincipal UserDetails userDetails
String username = userDetails.getUsername(); // 가능
Long id = userDetails.getId();               // 컴파일 에러 - id 필드 없음

// CustomUserDetails → id, email 등 추가 필드 접근 가능
@AuthenticationPrincipal CustomUserDetails userDetails
Long id = userDetails.getId();       // 가능
String email = userDetails.getEmail(); // 가능
```

`CustomUserDetails` 구현 예시:

```java
public class CustomUserDetails implements UserDetails {

    private Long id;
    private String email;
    private String password;
    private Collection<? extends GrantedAuthority> authorities;

    // UserDetails 필수 구현 메서드
    @Override
    public String getUsername() { return email; }

    @Override
    public String getPassword() { return password; }

    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() { return authorities; }

    // 추가 필드 접근자
    public Long getId() { return id; }
    public String getEmail() { return email; }
}
```

Controller에서의 사용:

```java
@GetMapping("/me")
public UserResponse getMe(@AuthenticationPrincipal CustomUserDetails userDetails) {
    Long userId = userDetails.getId();
    return userService.getUser(userId);
}
```

## 3. 권한 기반 접근 제어

### Method Security

Spring Security는 URL 패턴 기반 외에도 메서드 단위로 권한을 제어할 수 있다. URL 패턴으로 처리하기 어려운 복잡한 비즈니스 보안 요구사항에 활용한다.

### `@PreAuthorize`

메서드 실행 전에 권한을 검증한다. 조건을 만족하지 않으면 메서드 자체가 실행되지 않는다.

```java
@PreAuthorize("hasAuthority('ADMIN')")
public UserListResponse getUsers() { ... }
```

### `@PostAuthorize`

메서드 실행 후 반환된 결과를 기반으로 권한을 검증한다. `returnObject`로 반환값에 접근할 수 있다.

```java
@PostAuthorize("returnObject.userId == authentication.principal.id")
public UserResponse getUser(Long userId) { ... }
```

#### `@PostAuthorize` 사용 시 전제 조건

`authentication.principal.id`는 기본 `UserDetails`에 없는 필드다. 위 코드가 동작하려면 `CustomUserDetails`에 `id` 필드가 정의되어 있어야 하고, 반환 객체인 `UserResponse`에도 `userId` 필드가 있어야 한다.

| 조건 | 이유 |
|------|------|
| `CustomUserDetails.id` 필드 존재 | `authentication.principal.id` 접근 가능 |
| `UserResponse.userId` 필드 존재 | `returnObject.userId` 접근 가능 |

```java
public class CustomUserDetails implements UserDetails {
    private Long id; // 없으면 런타임 에러 발생
    public Long getId() { return id; }
}

public class UserResponse {
    private Long userId; // 없으면 런타임 에러 발생
}
```
