# 파일 업로드(File Upload)
- 사용자가 선택한 파일을 서버로 전송하여 저장하는 기능
- Spring에서는 `MultipartFile` 객체를 통해 업로드 파일을 처리
- 파일 정보(파일명, 크기, 타입 등)와 실제 파일 데이터를 함께 전달 가능
- 주로 이미지 업로드, 첨부파일 등록, 문서 관리 기능 등에 사용

### MultipartFile

- 업로드된 파일 정보를 담고 있는 객체
- 주요 메소드

| 메소드 | 설명 |
|----------|----------|
| getOriginalFilename() | 원본 파일명 조회 |
| getSize() | 파일 크기 조회 |
| getContentType() | MIME 타입 조회 |
| isEmpty() | 빈 파일 여부 확인 |
| transferTo() | 실제 파일 저장 |

---

## 단일 파일 업로드

### 단일 파일 업로드 처리

- `@RequestParam("file") MultipartFile file` 사용
- 요청받은 파일을 서버에 저장
- 업로드 결과를 JSON 형태로 응답

### 처리 과정

1. 클라이언트가 파일 전송
2. MultipartFile 객체 생성
3. 파일 유효성 검사
4. 서버 디렉토리에 저장
5. 저장 결과 반환

---

## 다중 파일 업로드

### 다중 파일 업로드 처리

- `List<MultipartFile>` 형태로 여러 파일 수신
- 반복문을 통해 파일별 저장 처리

```java
@RequestParam("files")
List<MultipartFile> files
```

### 특징

- 여러 파일을 한 번의 요청으로 처리 가능
- 업로드 성공 파일 정보를 리스트로 관리
- 업로드 개수와 파일 목록을 응답

---

## 파일 저장 경로 관리

### 업로드 경로 설정

```java
@Value("${file.upload-dir}")
```

- application.properties에 정의된 경로 사용
- 환경에 따라 업로드 위치 변경 가능
- 코드 수정 없이 설정 파일만 변경 가능

### Path 객체 사용

```java
Path uploadRoot
```

- 운영체제에 독립적인 경로 처리
- 파일 저장 위치를 안전하게 관리

## 디렉토리 생성

### Files.createDirectories()

```java
Files.createDirectories(uploadPath);
```

- 업로드 폴더가 없으면 자동 생성
- 이미 존재하면 예외 발생 없이 통과

예시

```text
uploads/
 ├─ single/
 └─ multiple/
```

## 파일 유효성 검사

### validateFile()

업로드 전 필수 검증 수행

#### 빈 파일 검사

```java
file.isEmpty()
```

- 내용 없는 파일 업로드 방지

#### 파일명 존재 여부 검사

```java
file.getOriginalFilename()
```

- 원본 파일명이 없는 경우 업로드 차단

### 검증 목적

- 잘못된 요청 차단
- 서버 오류 방지
- 데이터 무결성 유지

## 확장자 추출

### 파일 확장자 분리

```java
lastIndexOf(".")
```

예시

```text
image.png → .png
test.txt → .txt
```

### 특징

- 마지막 "." 기준으로 확장자 추출
- 점이 여러 개인 파일도 정상 처리

예시

```text
my.image.profile.png
```

→ `.png`

---

## UUID를 이용한 파일명 생성

### 파일명 중복 방지

```java
UUID.randomUUID()
```

### 예시

원본 파일

```text
profile.png
```

저장 파일

```text
6d2f4f4d6e7349f3a8f3c5f9d7f2c1ab.png
```

### 장점

- 파일명 충돌 방지
- 덮어쓰기 방지
- 보안 향상

## 파일 저장

### transferTo()

```java
file.transferTo(targetPath);
```

### 역할

- MultipartFile을 실제 디스크에 저장
- 서버 내부 지정 경로로 파일 복사

저장 흐름

```text
Client
   ↓
MultipartFile
   ↓
transferTo()
   ↓
Server Disk
```
