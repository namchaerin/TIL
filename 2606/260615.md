# JPQL - Group Function

JPQL의 그룹 함수는 SQL과 동일하게 사용할 수 있다.

## 지원 함수

- COUNT()
- SUM()
- AVG()
- MAX()
- MIN()

---

## 주의사항

### 1. 반환 타입

그룹 함수의 결과는 다음 타입으로 반환된다.

| 함수 결과 | 반환 타입 |
|-----------|-----------|
| 정수 값 | Long |
| 실수 값 | Double |

예시

```java
Long count = query.getSingleResult();
Double avg = query.getSingleResult();
```

---

### 2. 조회 결과가 없을 경우

| 함수 | 반환값 |
|--------|--------|
| COUNT | 0 |
| SUM | null |
| AVG | null |
| MAX | null |
| MIN | null |

따라서 COUNT를 제외한 함수는 기본형(long, double) 대신 Wrapper 타입(Long, Double)을 사용하는 것이 안전하다.

```java
Long sumPrice = query.getSingleResult();
```

❌ 위험

```java
long sumPrice = query.getSingleResult(); // NPE 발생 가능
```

---

### 3. HAVING 절 파라미터 타입

그룹 함수 결과가 Long 또는 Double로 반환되므로 HAVING 절 비교값도 동일한 타입을 사용해야 한다.

```java
long minPrice = 50000L;
```

```java
HAVING SUM(m.menuPrice) >= :minPrice
```

---

## COUNT

특정 조건에 해당하는 데이터 개수 조회

```jpql
SELECT COUNT(m.menuPrice)
FROM Menu m
WHERE m.categoryCode = :categoryCode
```

반환 타입

```java
Long
```

---

## SUM

특정 조건의 합계 조회

```jpql
SELECT SUM(m.menuPrice)
FROM Menu m
WHERE m.categoryCode = :categoryCode
```

반환 타입

```java
Long
```

조회 결과가 없으면

```java
null
```

반환

---

## GROUP BY

카테고리별 메뉴 가격 합계 조회

```jpql
SELECT m.categoryCode, SUM(m.menuPrice)
FROM Menu m
GROUP BY m.categoryCode
```

결과

```text
카테고리코드 | 가격합계
```

---

## HAVING

그룹화된 결과에 조건 적용

```jpql
SELECT m.categoryCode, SUM(m.menuPrice)
FROM Menu m
GROUP BY m.categoryCode
HAVING SUM(m.menuPrice) >= :minPrice
```

예시

```java
Long minPrice = 50000L;
```

가격 합계가 50,000원 이상인 카테고리만 조회한다.
