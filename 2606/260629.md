# 📓 개발 일지 (Day 2)

## 학습 주제
댓글 API 검증 및 최신 브랜치 반영, 테스트 환경 문제 분석

## 학습 내용
- 댓글 API(GET, POST, PATCH, DELETE)의 인증, 권한, 입력값 검증 정책을 코드와 테스트 케이스를 통해 확인하였다.
- `git fetch`로 최신 `dev` 브랜치를 확인한 후 추가된 변경사항을 Rebase하여 최신 기준으로 브랜치를 정리하였다.
- Worktree 환경에서 테스트 수행 시 전체 테스트가 실패하는 현상을 분석하였다.
- 스택트레이스를 확인한 결과 `jwt.expiration` 환경변수가 비어 있어 `NumberFormatException`이 발생하며 애플리케이션 컨텍스트가 생성되지 않은 것이 원인이었다.
- 실제 `application-secret.env` 파일을 적용하여 테스트를 다시 수행한 결과 모든 테스트가 정상적으로 통과하는 것을 확인하였다.

## 학습한 내용 요약
- Rebase 이후에는 `git fetch`를 통해 최신 브랜치 기준을 확인한 후 작업하는 것이 충돌을 줄이는 데 도움이 된다.
- 테스트 실패가 반드시 코드 문제는 아니며, 환경변수 및 실행 환경까지 함께 확인하는 것이 중요하다.
- `git push --force-with-lease`를 사용하면 다른 사람의 변경사항을 보호하면서 Rebase 결과를 안전하게 Push할 수 있다.
