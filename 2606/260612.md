# 1. JPQL

## 1. JPQL (Java Persistence Query Language)

### 1.1 JPQL 개념

> 엔티티 객체를 대상으로 조회하는 객체 지향 쿼리 언어

- SQL과 유사한 문법 사용
- 테이블이 아닌 엔티티 기준으로 작성
- 특정 DBMS에 종속되지 않음
- JPA가 DB에 맞는 SQL로 변환하여 실행

---

### 1.2 JPQL 실행 과정

1. JPQL 문자열 작성
2. `createQuery()`로 Query 객체 생성
3. `getSingleResult()` 또는 `getResultList()` 실행

```java
String jpql = "SELECT m FROM Menu m";

TypedQuery<Menu> query =
        entityManager.createQuery(jpql, Menu.class);

List<Menu> result = query.getResultList();
```

---

## 2. Query 종류

### 2.1 TypedQuery

> 반환 타입이 명확할 때 사용

```java
TypedQuery<Menu> query =
        entityManager.createQuery(jpql, Menu.class);
```

#### 특징

- 컴파일 시 타입 체크 가능
- 형변환 불필요
- 가장 많이 사용

---

### 2.2 Query

> 반환 타입이 명확하지 않을 때 사용

```java
Query query = entityManager.createQuery(jpql);
```

#### 특징

- Object 또는 Object[] 반환
- 형변환 필요

---

## 3. 조회 메소드

### 3.1 getSingleResult()

> 결과가 1건일 때 사용

```java
query.getSingleResult();
```

#### 특징

- 결과 1개 → 정상 반환
- 결과 없음 → 예외 발생
- 결과 여러 개 → 예외 발생

---

### 3.2 getResultList()

> 결과가 여러 건일 때 사용

```java
query.getResultList();
```

#### 특징

- List 반환
- 결과가 없으면 빈 List 반환

---

## 4. JPQL 조회 예제

### 4.1 단일 컬럼 조회

```java
SELECT m.menuName
FROM Menu m
WHERE m.menuCode = 8
```

---

### 4.2 단일 엔티티 조회

```java
SELECT m
FROM Menu m
WHERE m.menuCode = 8
```

---

### 4.3 다중 엔티티 조회

```java
SELECT m
FROM Menu m
```

---

## 5. 주요 연산자

### 5.1 DISTINCT

> 중복 제거

```java
SELECT DISTINCT m.categoryCode
FROM Menu m
```

---

### 5.2 IN

> 여러 값 조건 조회

```java
SELECT m
FROM Menu m
WHERE m.categoryCode IN (11, 12)
```

---

### 5.3 LIKE

> 문자열 검색

```java
SELECT m
FROM Menu m
WHERE m.menuName LIKE '%마%'
```

---

## 6. 파라미터 바인딩

### 6.1 이름 기준 바인딩 (권장)

```java
String jpql =
        "SELECT m FROM Menu m WHERE m.menuName = :menuName";

entityManager.createQuery(jpql, Menu.class)
             .setParameter("menuName", menuName)
             .getResultList();
```

#### 특징

- 가독성 좋음
- 유지보수 편리
- 실무에서 가장 많이 사용

---

### 6.2 위치 기준 바인딩

```java
String jpql =
        "SELECT m FROM Menu m WHERE m.menuName = ?1";

entityManager.createQuery(jpql, Menu.class)
             .setParameter(1, menuName)
             .getResultList();
```

#### 특징

- 위치 번호 사용
- 쿼리 변경 시 관리 어려움
- 실무에서는 거의 사용하지 않음
