# Mapping

## 1. Entity

- JPA에서 데이터베이스 테이블과 매핑되는 객체
- `@Entity` 어노테이션을 사용하여 엔티티로 지정
- 영속성 컨텍스트가 관리하는 객체
- 반드시 식별자(PK)를 가져야 함

### 주요 어노테이션

| 어노테이션 | 설명 |
|-----------|------|
| @Entity | 엔티티 클래스 지정 |
| @Table | 매핑할 테이블 지정 |
| @Id | 기본 키(PK) 지정 |
| @Column | 컬럼 매핑 |

### 특징

- DB 테이블과 1:1로 매핑
- 영속성 컨텍스트에 의해 관리됨
- CRUD 작업의 대상

---

## 2. Embedded

- 여러 필드를 하나의 값 객체(Value Object)로 묶어 관리하는 기능
- 공통 속성을 재사용하기 위해 사용

### 주요 어노테이션

| 어노테이션 | 설명 |
|-----------|------|
| @Embeddable | 임베디드 타입 정의 |
| @Embedded | 엔티티에 임베디드 타입 포함 |

### 예시

```java
@Embeddable
public class Price {

    private int regularPrice;
    private double discountRate;
    private int sellPrice;
}
```

```java
@Entity
public class Book {

    @Embedded
    private Price price;
}
```

### 특징

- 별도 테이블 생성 X
- 엔티티 테이블의 컬럼으로 함께 저장
- 관련 데이터를 하나의 객체로 관리 가능
- 비즈니스 로직을 값 객체 내부에 캡슐화 가능

### 장점

- 코드 재사용성 증가
- 객체지향적인 설계 가능
- 관련 필드 관리 용이

# Association Mapping (연관관계 매핑)

## 1. Association Mapping
### 1.1 개념
> 엔티티 간의 관계를 객체 참조를 통해 표현하는 것

DB는 FK(외래키)로 관계를 맺고, JPA는 객체 참조로 관계를 맺는다.

```text
DB
Category ← FK → Menu

JPA
Category ← 객체 참조 → Menu
```

---

### 1.2 연관관계 종류

| 관계 | 어노테이션 |
|--------|--------|
| N:1 | @ManyToOne |
| 1:N | @OneToMany |
| 1:1 | @OneToOne |
| N:M | @ManyToMany |

---

### 1.3 방향성

#### 단방향

```text
Menu → Category
```

한쪽 객체만 다른 객체를 참조

#### 양방향

```text
Menu ↔ Category
```

양쪽 객체가 서로를 참조

---

### 1.4 연관관계 주인

- 외래키(FK)를 관리하는 엔티티
- DB 반영 기준이 되는 객체
- `mappedBy`가 없는 쪽이 주인

---

## 2. ManyToOne (N:1)
> 여러 엔티티가 하나의 엔티티를 참조하는 관계

```text
여러 Menu → 하나의 Category
```

---

### 2.2 매핑

```java
@Entity
public class Menu {

    @Id
    private int menuCode;

    @ManyToOne
    @JoinColumn(name = "category_code")
    private Category category;
}
```

---

### 2.3 주요 어노테이션

| 어노테이션 | 설명 |
|-----------|------|
| @ManyToOne | 다대일 관계 지정 |
| @JoinColumn | FK 컬럼 지정 |

---

### 2.4 특징

- 가장 많이 사용하는 연관관계
- FK를 직접 관리
- 연관관계의 주인이 됨
- 단방향, 양방향 모두 가능

---

## 3. OneToMany (1:N)

### 3.1 개념

> 하나의 엔티티가 여러 엔티티를 참조하는 관계

```text
하나의 Category → 여러 Menu
```

---

### 3.2 단방향 매핑

```java
@Entity
public class Category {

    @Id
    private int categoryCode;

    @OneToMany
    @JoinColumn(name = "category_code")
    private List<Menu> menuList;
}
```

---

### 3.3 양방향 매핑

#### Category

```java
@OneToMany(mappedBy = "category")
private List<Menu> menuList;
```

#### Menu

```java
@ManyToOne
@JoinColumn(name = "category_code")
private Category category;
```

---

### 3.4 mappedBy

```java
@OneToMany(mappedBy = "category")
```

- 연관관계 주인이 아님
- FK를 직접 관리하지 않음
- 읽기 전용 역할

---

### 3.5 특징

- 컬렉션(List, Set) 사용
- 보통 ManyToOne과 함께 양방향으로 구성
- 실제 FK 관리는 ManyToOne 쪽이 담당
- OneToMany 단독 사용은 관리가 복잡하여 잘 사용하지 않음

---

### 3.6 정리

| 관계 | 주인 | FK 위치 |
|--------|--------|--------|
| ManyToOne | Many 쪽 | Many 테이블 |
| OneToMany | 주인 아님(mappedBy) | Many 테이블 |

예시

```text
Category (1)
      ↑
      │
Menu (N)

FK : MENU.CATEGORY_CODE
주인 : Menu (@ManyToOne)
```
