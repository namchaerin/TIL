# JPA 엔티티 생명주기(Entity Life Cycle)

## 01. 엔티티 생명주기 4가지 상태

| 상태 | 설명 |
|--------|--------|
| 비영속 (Transient) | 객체만 생성된 상태, 영속성 컨텍스트가 관리하지 않음 |
| 영속 (Managed) | 영속성 컨텍스트가 관리하는 상태 |
| 준영속 (Detached) | 영속성 컨텍스트와 분리된 상태 |
| 삭제 (Removed) | 삭제가 예약된 상태 |

---

## 02. 비영속 (Transient)

### 개념

- 객체만 생성된 상태
- 영속성 컨텍스트와 관련 없음

### 예시

```java
Menu newMenu = new Menu(...);
```

### 특징

```java
entityManager.contains(newMenu);
```

결과

```java
false
```

- 영속성 컨텍스트가 관리하지 않음
- DB와 연결되지 않음

---

## 03. 영속 (Managed)

### 개념

엔티티가 영속성 컨텍스트에 저장되어 관리되는 상태

### 영속 상태가 되는 방법

```java
entityManager.find(Menu.class, 1);

entityManager.persist(menu);
```

### 특징

- 1차 캐시에 저장됨
- 변경 감지(Dirty Checking) 가능
- 동일한 EntityManager에서 같은 PK 조회 시 같은 객체 반환

### 예시

```java
Menu menu1 = entityManager.find(Menu.class, 1);
Menu menu2 = entityManager.find(Menu.class, 1);

assertEquals(menu1, menu2);
```

---

## 04. 준영속 (Detached)

### 개념

영속성 컨텍스트가 더 이상 엔티티를 관리하지 않는 상태

### detach()

특정 엔티티만 준영속 상태로 변경

```java
entityManager.detach(menu);
```

### 특징

```java
menu.setMenuPrice(1000);

entityManager.flush();
```

- 수정해도 DB 반영 안됨
- 변경 감지 대상 아님

### merge()

준영속 엔티티를 다시 영속 상태로 변경

```java
entityManager.merge(menu);
```

### 특징

- 식별자(PK) 기준으로 엔티티 조회
- 값 병합 후 영속 객체 반환
- 변경 내용 DB 반영 가능

---

## 05. 영속성 컨텍스트 초기화(clear)

### 개념

영속성 컨텍스트 내부의 모든 엔티티를 준영속 상태로 변경

```java
entityManager.clear();
```

### 특징

- 1차 캐시 초기화
- 모든 엔티티 관리 해제

### 예시

```java
Menu menu1 = entityManager.find(Menu.class, 1);

entityManager.clear();

Menu menu2 = entityManager.find(Menu.class, 1);
```

결과

```java
menu1 != menu2
```

---

## 06. 영속성 컨텍스트 종료(close)

### 개념

영속성 컨텍스트 자체를 종료

```java
entityManager.close();
```

### 특징

- 모든 엔티티 준영속화
- EntityManager 사용 불가

### 예시

```java
entityManager.close();

entityManager.find(Menu.class, 1);
```

결과

```java
IllegalStateException 발생
```

---

## 07. 삭제 (Removed)

### 개념

엔티티 삭제 상태

```java
entityManager.remove(menu);
```

### 특징

- 영속성 컨텍스트에서 제거
- DB 삭제 예약 상태
- Commit 시 실제 삭제

### 예시

```java
transaction.begin();

entityManager.remove(menu);

entityManager.flush();
```

### flush()

```java
entityManager.flush();
```

- 영속성 컨텍스트 변경 내용을 DB와 동기화
- Commit 전에도 SQL 실행 가능
- Rollback 가능

---

## 08. 영속성 컨텍스트 상태 전이

```text
비영속 (Transient)
        |
        | persist(), find()
        ▼
영속 (Managed)
        |
        | detach(), clear(), close()
        ▼
준영속 (Detached)
        |
        | merge()
        ▼
영속 (Managed)
        |
        | remove()
        ▼
삭제 (Removed)
```

---

## 09. 핵심 정리

### 비영속

- 객체만 생성된 상태
- 영속성 컨텍스트가 관리하지 않음

### 영속

- 영속성 컨텍스트가 관리
- 1차 캐시 사용
- 변경 감지 가능

### 준영속

- 영속성 컨텍스트와 분리
- 수정해도 DB 반영 안됨
- merge()로 재영속화 가능

### 삭제

- remove() 호출 상태
- Commit 시 실제 삭제

### clear()

- 영속성 컨텍스트 초기화
- 모든 엔티티 준영속화

### close()

- 영속성 컨텍스트 종료
- EntityManager 사용 불가

### flush()

- 변경 내용을 DB에 즉시 반영
- Commit 전에도 실행 가능
- Rollback 가능
