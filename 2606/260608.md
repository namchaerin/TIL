# MyBatis Dynamic SQL

## 1. Dynamic SQL이란?

Dynamic SQL은 **조건에 따라 SQL을 동적으로 생성하는 기능**이다.

복잡한 검색 조건이나 수정 조건을 Java 코드에서 처리하지 않고 Mapper XML에서 관리할 수 있어 유지보수성이 높아진다.

---

## 2. `<if>`

조건이 참일 때만 SQL 구문을 추가한다.

### 예시

```xml
<if test="price gte 0 and price lte 10000">
    AND A.MENU_PRICE < #{ price }
</if>
```

### 사용 목적

- 조회 조건 동적 추가
- 파라미터 존재 여부 확인
- 조건별 SQL 분기 처리

### 결과

```sql
WHERE A.ORDERABLE_STATUS = 'Y'
AND A.MENU_PRICE < 10000
```

---

## 3. `<choose>`, `<when>`, `<otherwise>`

Java의 switch-case 문과 동일한 역할을 수행한다.

### 예시

```xml
<choose>
    <when test="value == '식사'">
        AND A.CATEGORY_CODE IN (4, 5, 6, 7)
    </when>

    <when test="value == '음료'">
        AND A.CATEGORY_CODE IN (8, 9, 10)
    </when>

    <otherwise>
        AND A.CATEGORY_CODE IN (11, 12)
    </otherwise>
</choose>
```

### 사용 목적

- 여러 조건 중 하나만 실행
- 복잡한 if 중첩 제거
- 조건 분기 처리

---

## 4. `<foreach>`

Java의 for문처럼 컬렉션을 반복 처리한다.

### 예시

```xml
<foreach collection="randomMenuCodeList"
         item="menuCode"
         open="("
         separator=","
         close=")">
    #{ menuCode }
</foreach>
```

### 사용 목적

- IN 절 생성
- 대량 데이터 조회
- 리스트 반복 처리

### 결과

```sql
WHERE MENU_CODE IN (1, 5, 7, 10)
```

### 주요 속성

| 속성 | 설명 |
|--------|--------|
| collection | 반복할 컬렉션 |
| item | 반복 시 사용할 변수명 |
| open | 시작 문자열 |
| separator | 구분자 |
| close | 종료 문자열 |

---

## 5. `<where>`

동적 조건에 맞게 WHERE 절을 자동 생성한다.

### 예시

```xml
<where>
    A.ORDERABLE_STATUS = 'Y'

    <if test="condition eq 'menuCode'">
        AND A.MENU_CODE = #{ value }
    </if>
</where>
```

### 특징

- WHERE 자동 생성
- 맨 앞 AND, OR 자동 제거
- 조건이 없으면 WHERE 생성 안 함

### 결과

```sql
WHERE A.ORDERABLE_STATUS = 'Y'
AND A.MENU_CODE = 10
```

### 사용 전

```xml
WHERE
AND MENU_CODE = #{value}
```

→ SQL 문법 오류

### 사용 후

```xml
WHERE MENU_CODE = #{value}
```

→ 정상 실행

---

## 6. `<trim>`

SQL 앞뒤 문자열을 자동으로 추가하거나 제거한다.

### 예시

```xml
<trim prefix="SET" prefixOverrides=",">
    <if test="name != null">
        , MENU_NAME = #{name}
    </if>

    <if test="categoryCode != null">
        , CATEGORY_CODE = #{categoryCode}
    </if>
</trim>
```

### 사용 목적

- UPDATE SET 절 동적 생성
- WHERE 태그보다 세밀한 제어

### 결과

```sql
SET MENU_NAME = '김치찌개',
    CATEGORY_CODE = 4
```

### 주요 속성

| 속성 | 설명 |
|--------|--------|
| prefix | 앞에 붙일 문자열 |
| prefixOverrides | 제거할 앞 문자열 |
| suffix | 뒤에 붙일 문자열 |
| suffixOverrides | 제거할 뒤 문자열 |

## 7. `<resultMap>`

조회 결과를 DTO 객체에 매핑한다.

### 예시

```xml
<resultMap id="menuResultMap" type="MenuDTO">
    <id property="code" column="MENU_CODE"/>

    <result property="name"
            column="MENU_NAME"/>

    <result property="price"
            column="MENU_PRICE"/>

    <result property="categoryCode"
            column="CATEGORY_CODE"/>

    <result property="orderableStatus"
            column="ORDERABLE_STATUS"/>
</resultMap>
```

### 사용 목적

- DB 컬럼 ↔ DTO 필드 연결
- 컬럼명과 필드명이 다를 때 매핑
- 복잡한 객체 관계 처리

### 예시

```java
private int code;
private String name;
```

↓

```sql
MENU_CODE
MENU_NAME
```

자동 매핑

## 8. `#{}` 파라미터 바인딩

PreparedStatement 방식으로 값을 바인딩한다.

### 예시

```xml
AND MENU_CODE = #{ code }
```

### 동작

```sql
AND MENU_CODE = ?
```

↓

```java
preparedStatement.setInt(1, code);
```

### 장점

- SQL Injection 방지
- 자동 타입 변환
- 가장 많이 사용하는 방식

---

## 9. CDATA

XML에서 특수문자를 사용할 수 있도록 해준다.

### 예시

```xml
<![CDATA[
AND A.MENU_PRICE < #{ price }
]]>
```

### 사용 이유

XML에서는 다음 문자를 그대로 사용할 수 없다.

```xml
<
>
<=
>=
```

따라서 CDATA로 감싸 XML 파싱 오류를 방지한다.

## 10. Mapper에서 사용된 조건 연산자

### 비교 연산자

| 표현식 | 의미 |
|--------|--------|
| gt | `>` |
| lt | `<` |
| gte | `>=` |
| lte | `<=` |
| eq | `==` |
| neq | `!=` |

### 예시

```xml
<if test="price gt 10000">
```

↓

```java
if(price > 10000)
```
