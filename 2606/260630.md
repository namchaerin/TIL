# 📓 개발 일지 (Day 3)

## 학습 주제
브랜치 푸시, Pull Request 생성 및 코드 품질 검증

## 학습 내용
- Rebase를 완료한 `feature/BE-recipe-comment` 브랜치를 `--force-with-lease` 옵션을 사용하여 안전하게 원격 저장소에 반영하였다.
- Push 전 `git log --oneline dev..HEAD`와 `git status`를 통해 댓글 관련 커밋만 포함되어 있는지와 작업 상태를 최종 점검하였다.
- 댓글 CRUD(API-307~310)에 대한 Pull Request를 생성하고, 작업 내용과 인증·권한 정책, 테스트 결과를 정리하여 리뷰어가 변경사항을 쉽게 파악할 수 있도록 작성하였다.
- Rebase 과정에서 제외한 중복 커밋(API-304)에 대한 내용을 PR에 함께 기록하여 변경 이력을 명확하게 공유하였다.
- PR 생성 후 권한 분기(작성자, 관리자), Soft Delete 처리, 예외 처리(401/403)가 의도한 대로 동작하는지 테스트 및 코드 리뷰를 통해 다시 확인하였다.

## 학습한 내용 요약
- Rebase 이후에는 `--force-with-lease`를 사용하여 안전하게 브랜치를 Push하는 것이 중요하다.
- PR에는 구현 내용뿐 아니라 변경 이력과 작업 배경을 함께 작성하면 리뷰 효율을 높일 수 있다.
- 코드 작성 이후에도 권한 처리, 예외 처리, 삭제 정책 등을 다시 검증하는 과정이 안정적인 코드 품질을 유지하는 데 도움이 된다.
