# Validation & Swagger

## Validation

### Validation이란

- 클라이언트가 전달한 요청 데이터가 올바른 형식인지 검증하는 기능
- 잘못된 데이터가 서버 로직으로 전달되는 것을 방지
- 주로 DTO에 검증 어노테이션을 사용

### 주요 Validation 어노테이션

| 어노테이션 | 설명 |
|-----------|-----------|
| @NotNull | null 값 허용 안 함 |
| @NotBlank | null, 빈 문자열, 공백 허용 안 함 |
| @Size | 문자열 길이 제한 |

예시

```java
@NotBlank
private String name;

@Size(min = 2, max = 20)
private String nickname;
```

---

### Validation 예외 처리

- Validation 검증 실패 시 `MethodArgumentNotValidException` 발생
- `@ExceptionHandler`를 이용해 예외를 처리할 수 있음

```java
@ExceptionHandler(MethodArgumentNotValidException.class)
```

### Validation 응답 예시

```json
{
  "code": "ERROR_CODE_0002",
  "description": "필수 값이 공백입니다.",
  "detail": "이름은 반드시 입력해야 합니다."
}
```

### Validation 적용 목적

- 잘못된 요청 데이터 차단
- 데이터 무결성 보장
- 예외 처리 일관성 유지

---

## Swagger(OpenAPI)

### Swagger란

- REST API 문서를 자동 생성하는 도구
- API 정보, 요청 방식, 파라미터, 응답값 등을 확인 가능
- API 테스트 기능 제공

### Swagger 사용 목적

- API 명세 자동화
- 프론트엔드와 백엔드 협업 지원
- API 테스트 및 검증

---

### @Tag

```java
@Tag(
    name = "user 관련 api",
    description = "회원 조회, 등록, 수정, 삭제 API"
)
```

- API 그룹 정의
- Swagger 화면에서 카테고리 역할 수행

---

### @Operation

```java
@Operation(
    summary = "회원 목록 조회",
    description = "전체 회원을 조회한다."
)
```

- API 기능 설명 작성
- Swagger 문서에 표시됨

---

### @ApiResponse

```java
@ApiResponse(
    responseCode = "200",
    description = "회원 목록 조회 성공"
)
```

- API 응답 상태 코드 설명

---

### Swagger 문서화 예시

```java
@Tag(name = "회원 API")
@RestController
public class UserController {

    @Operation(
        summary = "회원 조회",
        description = "회원 번호로 회원 정보를 조회한다."
    )
    @ApiResponses({
        @ApiResponse(
            responseCode = "200",
            description = "조회 성공"
        ),
        @ApiResponse(
            responseCode = "404",
            description = "회원 없음"
        )
    })
    @GetMapping("/users/{userNo}")
    public UserDTO findUser() {
        return null;
    }
}
```
