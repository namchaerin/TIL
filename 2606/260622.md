# Spring Security 설정

## 1. 클래스 설정

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity
@RequiredArgsConstructor
public class SecurityConfig {
```

### 어노테이션 설명

| 어노테이션 | 설명 |
|-----------|------|
| `@Configuration` | Spring 설정 클래스 등록 |
| `@EnableWebSecurity` | Spring Security 활성화 |
| `@EnableMethodSecurity` | `@PreAuthorize`, `@PostAuthorize` 등 메서드 권한 검사 활성화 |
| `@RequiredArgsConstructor` | final 필드에 대한 생성자 자동 생성 |

---

## 3. PasswordEncoder 등록

```java
@Bean
public PasswordEncoder passwordEncoder(){
    return new BCryptPasswordEncoder();
}
```

### 역할
- 사용자 비밀번호 암호화
- BCrypt 해시 알고리즘 사용

### 사용 예시

```java
String encoded = passwordEncoder.encode(password);
boolean result = passwordEncoder.matches(rawPassword, encodedPassword);
```

## 5. CSRF 비활성화

```java
http.csrf(AbstractHttpConfigurer::disable)
```

### 의미
- CSRF(Cross Site Request Forgery) 보호 기능 비활성화

### 이유
JWT 기반 REST API는 세션을 사용하지 않으므로 일반적으로 비활성화

---

## 6. 세션 사용 안 함 (JWT 방식)

```java
.sessionManagement(session ->
    session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
```

### 의미

```text
STATELESS = 서버가 세션을 저장하지 않음
```

### 특징

- 로그인 상태를 서버가 기억하지 않음
- 요청마다 JWT 토큰 검증
- 완전한 REST API 구조

---

## 7. 인가 설정

```java
.authorizeHttpRequests(auth ->
    auth.anyRequest().permitAll())
```

### 의미

```text
모든 요청 허용
```

현재는 인증 없이 모든 API 접근 가능

### 실제 운영 예시

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/auth/**").permitAll()
    .requestMatchers("/admin/**").hasRole("ADMIN")
    .anyRequest().authenticated()
)
```

---

## 8. SecurityFilterChain 반환

```java
return http.build();
```

### 역할
- 설정된 SecurityFilterChain 생성
- Spring Security에 등록

# 현재 보안 정책 정리

```text
CSRF         : 비활성화
세션          : 사용 안 함 (STATELESS)
인증 방식      : JWT 예정
비밀번호 암호화  : BCrypt
인가 정책      : 모든 요청 허용
메서드 보안     : 활성화
```

# 현재 동작 흐름
```
Client
↓
Request
↓
Spring Security
↓
permitAll()
↓
Controller
```
※ 아직 JWT 인증 필터가 등록되지 않아
모든 요청이 인증 없이 통과됨
