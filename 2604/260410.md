# 생성자 함수, 배열 고차 함수 및 프로토타입 기반 상속

---
## 1. 학습내용

### **[생성자 함수와 인스턴스 생성]**
* **생성자 함수 정의**: `new` 연산자와 함께 호출하여 동일한 구조의 객체를 여러 개 생성할 수 있는 함수이며, 관례적으로 대문자로 시작하는 파스칼 케이스를 사용한다.
* **인스턴스 생성 과정**: `new` 호출 시 암묵적으로 빈 객체가 생성되어 `this`에 바인딩되고, 프로퍼티 초기화 과정을 거쳐 완성된 객체(`this`)가 반환된다.
* **안전장치**: `new` 없이 호출할 경우 `this`가 전역 객체를 가리키는 문제를 방지하기 위해 `new.target` 등을 활용하여 안전하게 인스턴스를 생성한다.

### **[배열과 고차 함수 활용]**
* **배열 기본 조작**: 인덱스를 통해 요소에 접근하며 `push`, `pop`, `unshift`, `shift`로 요소를 추가 및 제거하고, `splice`와 `slice`를 통해 배열을 수정하거나 추출한다.
* **배열 순회 및 탐색**: `forEach`로 배열 전체를 순회하며, `indexOf`와 `includes`를 통해 특정 요소의 위치나 존재 여부를 확인한다.
* **고차 함수**:
    * `map`: 배열의 모든 요소를 가공하여 새로운 배열을 생성한다.
    * `filter`: 특정 조건을 만족하는 요소만 추출하여 새로운 배열을 생성한다.
    * `reduce`: 배열의 모든 요소를 순회하며 하나의 누적된 결과값을 산출한다.
    * `sort`: 콜백 함수를 기준으로 배열을 정렬하며 원본 배열을 직접 변경한다.
    * `find`, `some`, `every`: 조건에 맞는 요소를 검색하거나 전체/일부의 조건 만족 여부를 판별한다.

### **[프로토타입 상속]**
* **프로토타입 기반 상속**: 모든 객체는 `[[Prototype]]`이라는 연결고리를 가지며, 이를 통해 부모 객체의 프로퍼티나 메서드를 상속받아 사용할 수 있다.
* **메모리 효율성**: 메서드를 인스턴스마다 생성하지 않고 프로토타입에 정의함으로써 메모리 낭비를 줄이고 자원을 공유한다.

## 2. 느낀 점
자바스크립트의 프로토타입 기반 상속이 Java의 클래스 상속과는 다른 유연함을 가지고 있음을 알게 되었으며, 특히 고차 함수를 활용한 데이터 가공 방식이 매우 직관적이라는 점을 느꼈다.

## 3. 활용 방안
데이터 리스트를 처리할 때 복잡한 반복문 대신 고차 함수를 적극 사용하여 코드의 가독성을 높이고, 생성자 함수와 프로토타입을 조합하여 메모리 효율적인 객체 지향 설계를 적용한다.
