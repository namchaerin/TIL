# Router
## Next.js 클라이언트 컴포넌트 전용 훅 정리
> Next.js의 App Router 환경에서 클라이언트 컴포넌트('use client') 작성 시 활용되는 next/navigation 패키지의 주요 훅 4가지에 대해 학습하였다.

### 1. useRouter
프로그래밍 방식으로 페이지 경로를 변경하거나 내비게이션 동작을 제어할 때 사용한다.

* 주요 기능: `router.push('/path')`를 통해 특정 페이지로 이동하거나, `router.replace()`, `router.back()`, `router.forward()` 등의 메서드를 호출할 수 있다.
* 특징: 단순한 링크 이동을 넘어, 특정 로직(예: 로그인 성공, 폼 제출 완료) 실행 후 페이지를 리다이렉트해야 할 때 필수적이다.

### 2. usePathname
현재 브라우저에 표시된 URL의 경로명(Pathname)을 읽어올 때 사용한다.
* 주요 기능: URL에서 도메인과 쿼리 스트링을 제외한 순수 경로 문자열을 반환한다.
예: `https://example.com` → `/shop/items` 반환
* 특징: 현재 위치한 페이지에 따라 네비게이션 바의 메뉴를 강조(Active 상태)하는 등 경로 기반 UI 로직을 구현할 때 활용된다.

### 3. useParams
URL 구조 내에 정의된 동적 경로 파라미터(Dynamic Segments) 값을 읽어올 때 사용한다.
* 주요 기능: 폴더 구조가 `app/post/[slug]/page.tsx`일 때, 현재 URL이 `/post/hello-world`라면 `{ slug: 'hello-world' }` 형태의 객체를 반환한다.
* 특징: 서버 컴포넌트가 아닌 클라이언트 컴포넌트 내부에서 현재 페이지의 식별 정보를 즉시 참조해야 할 때 유용하다.

### 4. useSearchParams
URL의 쿼리 스트링(Search Parameters)을 읽을 때 사용한다.
* 주요 기능: `?key=value` 형태의 데이터에 접근할 수 있게 해준다. `URLSearchParams` 인터페이스를 사용하여 특정 키의 값을 가져온다.
예: `searchParams.get('page')`
* 특징: 검색 필터, 정렬 조건, 페이지네이션 등 페이지의 세부 상태를 URL로 관리할 때 필수적으로 사용된다.


```
⚠️ 주의사항
위 훅들은 모두 클라이언트 컴포넌트에서만 동작한다. 
따라서 해당 훅을 사용하는 파일 최상단에는 반드시 'use client' 지시어가 선언되어 있어야 한다.
```
