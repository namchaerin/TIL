## 리액트 핵심 동작 원리 분석 및 적용

### 1. 선언적 UI(Declarative UI)와 상태 기반 렌더링
리액트는 DOM을 직접 조작하여 화면을 수정하는 명령형 방식이 아닌, UI의 상태를 정의하고 데이터에 따라 화면이 자동으로 변화하는 **'선언적 프로그래밍'** 방식을 지향한다.

* **이론적 적용:** "코멘트 등록 시 UI 노드를 생성하여 리스트에 추가하라"는 단계별 명령 대신, "현재 코멘트 데이터(State)를 기반으로 리스트 UI를 그려라"고 선언적으로 코드를 작성하였다.
* **구현 결과:** 사용자가 코멘트를 입력하고 저장했을 때, 개발자가 직접 화면을 수정하는 코드를 작성하지 않아도 `setComments`를 통해 데이터(State)만 변경하면 리액트가 이를 감지하여 UI를 자동으로 다시 그려주는(Re-render) 메커니즘을 확인하였다.

### 2. 컴포넌트 생명주기(Lifecycle)와 비동기 데이터 동기화
컴포넌트가 화면에 나타나고 사라지는 과정에서 특정 시점에 필요한 로직을 실행하는 것은 리액트의 핵심 기능 중 하나이다.

* **이론적 적용:** 페이지가 처음 로드되는 시점(Mount)에 맞춰 서버의 데이터를 가져오기 위해 `useEffect`를 활용하였다. 이는 화면이 먼저 사용자에게 보여진 후, 비동기로 서버 데이터를 호출하여 사용자 경험(UX)을 최적화하는 방식이다.
* **구현 결과:** `musicId`라는 고유 값이 변경될 때마다 해당 시점에 맞춰 서버에 새로운 데이터를 요청하고, 이를 리액트의 상태와 동기화함으로써 동적 페이지 전환 시에도 중단 없는 데이터 처리가 가능함을 학습하였다.
