# 자바: 메서드 호출 구조 및 클래스 활용 정리

### 1. 메서드 호출 방식
#### (1) 비정적 메서드 (Non-Static Method)
* 반드시 `new` 연산자로 객체를 생성한 후, 참조 변수를 통해 호출한다.
* 호출 예시: `참조변수명.메서드명();`

#### (2) 정적 메서드 (Static Method)
* 객체 생성 없이 클래스 이름을 통해 직접 호출한다.
* 호출 예시: `클래스명.메서드명();`



### 2. 클래스 설계 및 메서드 작성 (Calculator.java)
다른 클래스에서 공통으로 사용할 수 있는 계산 기능을 담은 클래스를 설계한다.
```java
public class Calculator {
    
    // Non-Static 메서드: 두 수 중 작은 값을 반환
    public int minNumberOf(int a, int b) {
        return (a < b) ? a : b;
    }

    // Static 메서드: 두 수 중 큰 값을 반환
    public static int maxNumberOf(int a, int b) {
        return (a > b) ? a : b;
    }
}
```

### 3. 외부 클래스 활용 및 Import 구조 (Main.java)
다른 패키지 혹은 분리된 클래스의 기능을 가져올 때는 import 문을 사용한다.

* **import 선언** `import 패키지경로.클래스명;`
패키지 선언 아래, 클래스 선언 위에 작성한다.

```java
import com.section01.method.Calculator; // Calculator 클래스 경로 지정

public class Main {
    public static void main(String[] args) {
        
        // 1. 정적(Static) 메서드 호출
        // 객체 생성 없이 '클래스명'으로 직접 접근
        int max = Calculator.maxNumberOf(10, 20);
        System.out.println("Max Value: " + max);

        // 2. 비정적(Non-Static) 메서드 호출
        // 반드시 'new'를 통해 객체를 생성해야 메모리에 할당됨
        Calculator calc = new Calculator();
        int min = calc.minNumberOf(10, 20);
        System.out.println("Min Value: " + min);
    }
}
```

### 4. 동작 구조 정리
- **Static (정적)**: 프로그램 실행 시 메모리(Static 영역)에 자동으로 로드되어 언제든 호출 가능한 상태가 된다.
- **Non-Static (인스턴스)**: 실행 중 new 연산자를 만날 때 메모리(Heap 영역)에 공간을 할당받으며, 객체가 존재할 때만 기능을 수행한다.
