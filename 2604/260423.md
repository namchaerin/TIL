# Zustand

## 1. Zustand란?
Zustand는 리액트 환경에서 전역 상태를 관리하기 위한 작고 빠르며 확장 가능한 라이브러리다. 특정 프레임워크에 종속되지 않는 유연함을 갖추고 있으며, 단순한 객체 형태로 스토어를 구성하여 복잡한 설정 없이도 강력한 상태 관리 기능을 제공한다.



---

## 2. Zustand를 사용하는 이유

* **직관적인 상태 변경:** `set` 함수 하나로 상태를 업데이트할 수 있어 코드의 가독성이 높다.
* **불필요한 리렌더링 방지:** 컴포넌트에서 필요한 상태(`user`나 `loading` 등)만 골라서 구독할 수 있어 효율적이다.
* **비동기 로직 내장:** 별도의 미들웨어 없이 스토어 안에서 `async/await`를 사용하여 데이터 페칭 로직을 직접 작성할 수 있다.
* **전역 접근성:** 어떤 컴포넌트에서도 `useUserStore`를 호출하는 것만으로 유저 정보에 접근하거나 로그아웃 액션을 실행할 수 있다.

---

## 3. Zustand 사용 방법

### ① 스토어 정의
작성한 코드는 유저 정보(`user`)와 로딩 상태(`loading`)를 전역으로 관리하며, 이를 제어하는 동기/비동기 함수들을 포함한다.

```javascript
import { create } from "zustand";

export const useUserStore = create((set) => ({
    // 1. 초기 상태값 (State)
    user: null,
    loading: false,

    // 2. 동기 액션 (Actions)
    login: (userData) => set({ user: userData }),
    logout: () => set({ user: null }),
    setUser: (user) => set({ user }),

    // 3. 비동기 액션 (Async Actions)
    fetchUser: async () => {
        set({ loading: true }); // 로딩 시작
        try {
            const res = await fetch('[https://jsonplaceholder.typicode.com/users/1](https://jsonplaceholder.typicode.com/users/1)');
            const data = await res.json();
            set({ user: data }); // 데이터 성공 시 저장
        } catch (error) {
            console.error("유저 정보 로드 실패:", error);
        } finally {
            set({ loading: false }); // 통신 종료 후 로딩 해제
        }
    }
}));
```

### ② 컴포넌트에서 활용
컴포넌트에서는 다음과 같이 필요한 데이터와 함수를 추출하여 사용한다.
```js
function UserProfile() {
    // 필요한 데이터만 선택하여 사용 (성능 최적화)
    const { user, loading, fetchUser, logout } = useUserStore();

    if (loading) return <div>로딩 중...</div>;

    return (
        <div>
            {user ? (
                <>
                    <h1>안녕하세요, {user.name}님!</h1>
                    <button onClick={logout}>로그아웃</button>
                </>
            ) : (
                <button onClick={fetchUser}>사용자 정보 가져오기</button>
            )}
        </div>
    );
}
```
