# 이벤트(Event)

---
## 1. 학습 내용
### [이벤트 핸들러 등록 방법]
* **이벤트 핸들러 어트리뷰트:** HTML 태그에 `onclick="..."`과 같이 직접 작성하는 방식. 유지보수가 어려워 현대적 방식에서는 권장되지 않는다.
* **이벤트 핸들러 프로퍼티:** 자바스크립트에서 요소의 `onclick` 프로퍼티에 함수를 할당한다. HTML과 코드를 분리할 수 있지만, 하나의 이벤트에 하나의 함수만 연결할 수 있다는 단점이 있다.
* **addEventListener (권장):** `element.addEventListener('click', 함수)` 형태로 사용한다. 동일한 이벤트에 여러 개의 핸들러를 등록할 수 있고, 이벤트를 정교하게 제어할 수 있는 표준 방식.

### [이벤트 객체와 `this`]
이벤트가 발생하면 브라우저는 사건의 상세 정보를 담은 **이벤트 객체(e)**를 생성하여 핸들러의 첫 번째 인자로 전달한다.

* **e.target:** 실제로 클릭이나 입력이 일어난 **가장 안쪽의 요소**.
* **e.currentTarget:** 이벤트 리스너가 **실제로 부착된 요소**.
* **this 키워드:** 일반 함수 내부의 `this`는 `e.currentTarget`과 같지만, **화살표 함수** 내부의 `this`는 상위 스코프(보통 window)를 가리키므로 요소를 참조할 때는 `e.currentTarget`을 사용하는 것이 가장 안전하다.



### [이벤트 전파와 위임]
이벤트는 발생한 지점에서 멈추지 않고 부모 요소로 전달되는 특성이 있다.

* **이벤트 버블링:** 하위 요소에서 발생한 이벤트가 연못의 파문처럼 최상위 요소까지 전달되는 현상.
* **이벤트 위임 (Delegation):** 버블링을 활용하여, 많은 자식 요소 각각에 이벤트를 거는 대신 **부모 요소에 단 하나의 핸들러만 등록**하여 자식들의 이벤트를 관리하는 기법이다.



### [이벤트 동작 제어]
브라우저의 기본 동작이나 이벤트의 흐름을 인위적으로 멈출 수 있다.

* **e.preventDefault():** 태그의 기본 동작(예: 링크 이동, 폼 제출 시 페이지 새로고침)을 막는다. 주로 데이터 유효성 검사 시 사용한다.
* **e.stopPropagation():** 이벤트가 부모 요소로 전파(버블링)되는 것을 막는다. (예: 모달의 내부 콘텐츠를 클릭했을 때 배경 클릭 이벤트가 발생하는 것을 방지)

### [주요 이벤트 종류]
| 종류 | 대표적인 이벤트 | 특징 |
| :--- | :--- | :--- |
| **마우스** | `click`, `mousedown`, `mouseup`, `contextmenu` | 클릭, 버튼 누름/뗌, 우클릭 제어 등 |
| **키보드** | `keydown`, `keyup` | 사용자가 입력한 키 값(`e.key`)이나 물리적 코드(`e.code`) 확인 |
| **폼(Form)** | `submit`, `focus`, `blur`, `input` | 서버 제출 시 검증, 입력 포커스 상태, 실시간 입력 값 변화 감지 |

## 2. 느낀 점
이벤트는 단순한 클릭 처리뿐 아니라, 웹 페이지의 모든 상호작용을 구성하는 핵심 요소라는 점이 인상적이었다.

## 3. 활용 방안
이벤트 위임과 addEventListener를 활용해 유지보수가 쉽고 성능이 좋은 인터랙티브 UI를 구현할 수 있다.
