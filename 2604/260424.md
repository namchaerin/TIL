# Zustand Persist 및 TanStack Query Provider

## 1. Zustand Persist (상태 유지 미들웨어)
Zustand Persist는 스토어의 상태를 브라우저 저장소(localStorage, sessionStorage 등)에 자동으로 저장하고, 페이지를 새로고침했을 때 해당 상태를 다시 복구해주는 미들웨어다.

#### 사용 이유
* **데이터 보존:** 사용자 로그인 정보, 다크 모드 설정, 장바구니 리스트 등 새로고침 후에도 유지되어야 하는 데이터를 관리하기 위함이다.
* **사용자 경험(UX) 향상:** 사용자가 이전에 작업하던 상태를 그대로 유지하여 서비스의 연속성을 제공한다.

#### 사용 방법
`persist` 함수로 스토어 설정을 감싸고, `name` 속성을 통해 저장소에서 사용할 키(Key) 이름을 지정한다.

```javascript
import { create } from "zustand";
import { persist } from "zustand/middleware";

export const usePersistStore = create(
    persist(    // localStorage 에 저장하는 기능
        // 첫 번째 인자
        (set) => ({
            theme: 'light',
            toggleTheme: () => 
                set((state) => ({theme: state.theme === 'light' ? 'dark' : 'light'}))
        }),
        // 두 번째 인자
        {
            name: 'theme-storage'   // 로컬 스토리지에 저장될 키 이름
        }
    )
)
```

## 2. TanStack Query Provider (데이터 페칭 관리자)

TanStack Query(구 React Query)를 사용하기 위해 애플리케이션의 최상단에서 서버 상태(Server State)를 관리하는 컨텍스트를 제공하는 컴포넌트다. React의 Context API를 기반으로 동작하며, 하위 컴포넌트 어디서든 서버 데이터를 조회하고 관리할 수 있게 돕는다.

#### 사용 이유
* **QueryClient 공유:** 애플리케이션 전역에서 동일한 `QueryClient` 인스턴스를 공유하여 캐싱된 데이터에 접근하고 서버 통신 로직을 통합 관리한다.
* **서버 상태 동기화:** 캐시된 데이터의 신선도(staleTime)를 체크하고, 자동으로 배경에서 데이터를 업데이트하거나 가비지 컬렉션을 수행하는 등 복잡한 서버 상태 관리 기능을 활성화한다.
* **DevTools 연동:** Provider 내부에 DevTools를 설정하여 현재 진행 중인 쿼리 상태나 캐시 데이터를 시각적으로 디버깅할 수 있다.

#### 사용 방법
`QueryClient` 인스턴스를 생성한 뒤, `QueryClientProvider`로 전체 앱(또는 상위 레이아웃)을 감싸준다.

```javascript
'use client'

import { QueryClient, QueryClientProvider } from "@tanstack/react-query"
import { useState } from "react"

export default function Providers({children}) {
    const [queryClient] = useState(() => new QueryClient({
        defaultOptions: {
            queries: {
                staleTime: 60 * 1000,   // 1분 동안은 다시 요청하지 않고, 캐시를 사용
            },
        },
    }))

    return (
        <QueryClientProvider client={queryClient}>
            {children}
        </QueryClientProvider>
    )
}
```
