# DOM(Document Object Model) API 활용

## 1. 학습내용

### **[DOM 요소 선택 및 컬렉션 특성]**
* **선택 메서드**: `querySelector`와 `querySelectorAll`은 CSS 선택자를 인자로 받아 일치하는 요소 노드 또는 `NodeList`를 반환한다.
* **동적 컬렉션(Live Collection)**: `getElementsByClassName` 등이 반환하는 `HTMLCollection`은 DOM의 변화를 실시간으로 반영하므로, 반복문 사용 시 인덱스 불일치 문제를 방지하기 위해 배열로 변환하여 처리한다.

### **[노드 탐색 (Node Traversal)]**
* **요소 노드 탐색**: 텍스트 노드나 주석 노드를 제외하고 실제 HTML 태그만 탐색할 때는 `children`, `firstElementChild`, `nextElementSibling` 프로퍼티를 사용한다.
* **계층 구조 참조**: `parentNode`를 통해 상위 노드에 접근하며, DOM 트리의 부모, 자식, 형제 관계를 이용해 특정 노드에서 타겟 노드로 이동한다.

### **[노드 조작 및 성능 최적화]**
* **텍스트 제어**: `nodeValue`는 텍스트 노드에만 유효하나, `textContent`는 요소 노드에서 모든 자식 텍스트를 직관적으로 읽고 쓸 수 있다.
* **DOM 삽입 및 성능**: `createElement`와 `appendChild`로 요소를 추가하며, 다수의 노드를 삽입할 때는 `DocumentFragment`를 메모리상의 임시 컨테이너로 활용하여 브라우저의 리플로우(Reflow) 횟수를 1회로 단축한다.
* **위치 기반 삽입**: `insertAdjacentHTML`은 특정 요소를 기준으로 전후좌우 네 가지 위치(`beforebegin`, `afterbegin` 등)에 HTML 문자열을 파싱하여 삽입한다.

### **[어트리뷰트(Attribute)와 프로퍼티(Property)]**
* **상태 관리 차이**: HTML 어트리뷰트는 문서 내 초기 상태를 유지하고, DOM 프로퍼티는 사용자 입력에 의한 실시간 상태(예: input의 value)를 관리한다.
* **사용자 정의 데이터**: HTML의 `data-` 속성은 자바스크립트의 `dataset` 객체를 통해 접근 가능하며, 비표준 데이터를 요소에 저장하는 용도로 사용한다.

### **[CSS 제어]**
* **인라인 스타일**: 요소의 `style` 프로퍼티에 직접 접근하여 CSS 속성을 할당하며, 속성명은 카멜 케이스(camelCase)를 적용한다.
* **클래스 기반 제어**: `classList`의 `add`, `remove`, `toggle` 메서드를 사용하여 사전에 정의된 CSS 클래스를 동적으로 적용하고 관리한다.

## 2. 느낀 점
DOM API를 통해 HTML 문서를 객체 구조로 제어하는 구체적인 방법을 익혔다. 특히 라이브 컬렉션의 동작 방식과 DocumentFragment를 통한 성능 최적화 기법의 중요성을 확인했다.

## 3. 활용 방안
데이터 상태 변화에 따라 DOM을 업데이트할 때 직접적인 스타일 수정보다는 클래스 제어를 우선하며, 대규모 노드 생성 시 메모리 효율을 고려해 fragment를 사용한다.
