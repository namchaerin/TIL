# React - Hooks(2)
---

### useMemo
state를 변화시킬 때(update 시)마다 리렌더링이 이루어지는 과정에서 컴포넌트의 지역변수부터 return구문까지 재실행(리렌더링) 된다. 이는 state만 변화하고 싶은데 만약 **리렌더링 과정에서 불필요하게 동작하는 리소스가 있고 심지어 많은 리소스를 소모한다면 성능에 지대한 영향**을 끼칠 수도 있다.

**useMemo** hook를 활용하면 **캐싱(useMemo의 첫번째 인자에서 반환한 값)을 통해 불필요 작업이 리렌더링 시에 발생하지 않도록 처리할 수 있다.** 또한 useEffect처럼 의존성 배열에 지정한 값의 변화 시에만 캐싱 된 값이 재할당 되도록 진행 하여 코드의 성능 및 효율 향상에 도움을 줄 수 있다.

<useMemo 함수 구조>
```js 
const 변수명= useMemo(() => {    
    return useMemo로 캐싱 할 객체
    }, [useMemo가 리턴하는 객를 재할당 할 조건이 될 값]);
```

### useCallback
컴포넌트가 update 될 때마다 컴포넌트 내부에 지역변수로 선언한 함수가 있다면, 해당 함수는 매번 불필요하게 새로 생성된다. 만약 이 함수가 useEffect의 의존성 배열에서 사용된다면 함수와 관련 없는 이유로 컴포넌트가 리렌더링 될 때도 함수가 다시 정의 되며(함수도 객체의 일종으로 재할당 되어 주소값이 바뀜) useEffect가 매번 동작하게 된다.

함수 재할당이 불필요하게 계속 되어 리렌더링 시 변화로 간주될 때 **useCallback을 이용하여 해당 함수를 캐싱(memoization)** 할 수 있다. 캐싱되는 값은 useCallback의 첫번째 인자로 작성 된 익명함수이다.또한 useCallback의 의존성 배열을 사용하여 의존성 배열에 등록된 값의 변화가 생길 때만 함수가 재할당 되도록 할 수도 있다.

<useCallback함수의 구조>
```js
const 변수명= useCallback(
    useCallback으로 캐싱 할 함수, 
    [useCallback의 함수를 재할당 하게할 값]
    );
```

### useMemo vs useCallback
```
의존성 배열에 등록된 것이 리터럴 객체이고 재할당 되는 문제 때문에 useMemo를쓰고, 
반면에 의존성 배열에 등록된 것이 함수이고 함수의 재할당으로 문제가 된다면 useCallback을 사용하면 된다.
```

### useRef
useRef도 useMemo나 useCallback처럼 리렌더링에 지정 받지 않도록 따로 캐싱해 저장하는 개념이다. (변수 관리)
일반적으로는 DOM 요소를 저장해 놓고 다시 참조하기 위해 **DOM 요소를 캐싱**하는 개념으로 사용한다.(특정 DOM 요소를 선택하는 선택자와 같은 개념


useRef를 사용하여 **원하는 값을 리렌더링에 의해 지장 받지 않고 사용**할 수 있다. 
**useRef의 속성 current:** useRef의 인자로 전달한 초기 값을 담고 있으며 수정이 가능하다. (DOM 요소의 ref attribute를 통해 선언한 useRef가 DOM 요소를 저장하고 리렌더링에 의해 영향을 받지 않도록 할 수 있다.)


### 커스텀 훅 (Custom Hook)
리액트의 내장 훅(`useState`, `useEffect` 등)을 조합하여 복합적인 로직을 하나로 묶어 재사용 가능하게 만든 자바스크립트 함수다.

함수의 이름은 반드시 `use`로 시작해야 하며, 이를 통해 리액트 엔진이 훅의 규칙(최상위 호출 등)을 적용하도록 강제한다.

동일한 커스텀 훅을 여러 컴포넌트에서 호출하더라도 각 컴포넌트 내부의 상태(State)는 공유되지 않고 독립적으로 관리된다.

```js
// 커스텀 훅 정의
// 입력 로직만 따로 분리한 함수다.
function useInput(initialValue) {
    const [value, setValue] = useState(initialValue);

    const onChange = (e) => {
        setValue(e.target.value);
    };

    // 컴포넌트에서 사용할 상태값과 이벤트 핸들러를 반환한다.
    return { value, onChange };
}
```

```js
// 컴포넌트에서 적용
function App() {
    // 한 줄의 코드로 입력 로직을 가져와 사용한다.
    const nameInput = useInput('');
    const emailInput = useInput('');

    return (
        <div>
            {/* 스프레드 문법(...)을 쓰면 value와 onChange가 자동으로 연결된다. */}
            <input {...nameInput} placeholder="이름 입력" />
            <input {...emailInput} placeholder="이메일 입력" />
            
            <p>입력된 이름: {nameInput.value}</p>
        </div>
    );
}
```
