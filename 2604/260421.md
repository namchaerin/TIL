# 자바스크립트 비동기 프로그래밍 및 API 활용

## 1. 비동기 프로그래밍의 기초

비동기(Async) 및 넌블로킹(Non-blocking) 작업은 메인 흐름을 멈추지 않고 백그라운드에서 특정 작업들을 병렬적으로 처리하는 방식이다.

* **콜백 함수 (Callback Function)**
    * 비동기 작업을 처리할 때 가장 많이 사용하는 방식이다.
    * `setTimeout`과 같은 함수를 사용하여 작성 순서와 상관없이 특정 시간 후에 코드가 실행되도록 제어할 수 있다.
    * **콜백 지옥 (Callback Hell):** 특정 비동기 처리 이후 수행할 내용을 계속 중첩해서 정의하다 보면 가독성이 떨어지고 유지보수가 힘들어지는 패턴이 발생하는데, 이를 콜백 지옥이라 하며 지양해야 한다.

* **Promise (ES6)**
    * 콜백 지옥을 해결하고 비동기 통신 간의 순서를 쉽게 정하기 위해 도입되었다.
    * 미래의 어떤 시점에 결과를 제공하겠다는 '약속'을 반환하며, 세 가지 상태를 가진다: **대기(Pending)**, **이행(Fulfilled)**, **거부(Rejected)**.
    * 성공 시 `resolve`, 실패 시 `reject` 함수를 활용하며, `.then()`을 통한 메소드 체이닝으로 비동기 결과를 순차적으로 처리할 수 있다.

* **Async/Await (ES8)**
    * Promise 객체를 더 쉽고 동기식 코드처럼 작성할 수 있도록 추가된 문법이다.
    * `await` 키워드는 비동기 처리를 동기식으로 동작하게 하며, Promise의 `resolve`에 담긴 값을 직접 반환받을 수 있게 해준다.

## 2. API를 활용한 데이터 통신 (API)

* **Fetch API**
    * 대부분의 모던 브라우저에서 지원하는 Promise 기반의 비동기 통신 라이브러리다.
    * `fetch(url, [options])` 형태로 사용하며, 기본적으로 GET 메소드로 요청을 보낸다.
    * 응답 데이터는 `response.json()` 메소드를 사용하여 자바스크립트 객체로 파싱하여 사용한다. 단, 본문 내용(body)은 1회만 접근 가능하다는 점에 주의해야 한다.

* **Axios**
    * `fetch`와 달리 별도의 JSON 변환 과정(`json()`) 없이도 파싱된 데이터가 `data` 프로퍼티에 담겨 있어 편리하게 조회할 수 있는 라이브러리다.
