# React Component

---
## 1. 학습 내용

### [컴포넌트와 JSX]

- React는 UI를 컴포넌트 단위로 분리하여 구성한다.
- 컴포넌트는 함수 형태로 정의되며 JSX를 반환한다.
- JSX는 HTML처럼 보이지만 JavaScript 문법이며, 내부적으로 React.createElement로 변환된다.

### [JSX 특징]
- {} 안에서 변수, 함수 호출 등 JS 표현식 사용 가능
- class → className, 이벤트는 camelCase(onClick)
- style은 객체 형태로 전달
- 모든 태그는 반드시 닫아야 함
- 여러 요소는 Fragment(`<> </>`)로 감쌈

---

### [렌더링 구조]

- ReactDOM.createRoot().render()를 통해 컴포넌트를 DOM에 렌더링한다.
- 컴포넌트는 상태(state)나 props가 변경되면 다시 렌더링된다.
- setInterval을 통해 강제 렌더링도 가능하지만, 일반적으로는 상태 변경을 통해 UI를 갱신한다.

---

### [Props (데이터 전달)]

- Props는 부모 → 자식으로 전달되는 읽기 전용 데이터이다.
- 함수 매개변수 구조분해로 쉽게 사용할 수 있다.
- 기본값 설정이 가능하며, children으로 태그 내부 요소를 전달할 수 있다.

## 2. 느낀 점
Props를 통해 데이터가 부모에서 자식으로 단방향으로 흐르는 구조가 명확해서, 데이터 흐름을 추적하기 쉽다고 느꼈다.

## 3. 활용 방안
공통 버튼이나 입력 컴포넌트를 분리하고 Props로 동작을 주입하는 방식으로 재사용성을 높일 수 있을 것 같다.
